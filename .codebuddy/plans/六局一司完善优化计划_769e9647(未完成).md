## 产品概述

按渐进增强方案完善六局一司架构，分四阶段执行：命名方案统一 → gstack技能整合 → 插件安装 → 典级Skill实现。

## 核心功能

- **Phase 1**：命名方案统一，更新架构文档以匹配当前目录命名
- **Phase 2**：完善5个局的gstack整合包（尚融局、尚宣局、尚工局、尚教局、御鉴司）
- **Phase 3**：安装推荐插件（code-review、security-guidance等）
- **Phase 4**：创建63个典级Skill文件，补全架构层级

## 技术栈

- 配置文件：Markdown (.md)、JSON
- 目录结构：符合 OpenClaw 框架规范
- 技能整合：gstack技能包、superpowers技能
- 文件模板：司级SKILL.md标准格式

### Phase 4：典级Skill实现（2小时）

**目标**：创建63个典级SKILL.md文件

**目录结构**：

```
workspace/skills/{局名}/司级/{司级名}/典级/{典级名}/SKILL.md
```

**典级命名规则**：

- 格式：X典（2个字）
- 必须以"典"结尾
- 全局唯一

**典级分布**：

| 局级 | 司级 | 典级（每司3典） |
| --- | --- | --- |
| 尚融局 | 司投师 | 投典、资典、金典 |
| 尚工局 | 司具师 | 工典、具典、器典 |
| 尚宣局 | 司增师 | 增典、流典、粉典 |
| 尚机局 | 司试师 | 试典、测典、验典 |
| 尚宫局 | 司研师 | 研典、报典、析典 |
| 御鉴司 | 司安师 | 安典、防典、护典 |
| 御鉴司 | 司实师 | 实典、查典、核典 |
| 御鉴司 | 司证师 | 证典、规典、合典 |


**SKILL.md模板**：

```
---
description: [职责描述]。触发词：[关键词1]、[关键词2]
---

# [典级名]

## 基本信息
| 属性 | 值 |
|------|------|
| **名字** | [典级名] |
| **级别** | L5（典级） |
| **归属** | [局名] → [司级名] |
| **上级** | [司级名] |

## 核心职责
[一句话描述]

## 能力清单
| 能力项 | 描述 |
|--------|------|
| [能力1] | [描述] |

## 触发关键词
- "[关键词1]"
- "[关键词2]"
```
