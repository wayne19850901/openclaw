## 一、项目定位

### 1.1 Claw 是什么

Claw 是 **OpenClaw AI Agent 系统**的管理工具和开发工作区。

### 1.2 与 OpenClaw 的关系

```
┌─────────────────────────────────────────────────────┐
│                    Claw 项目                        │
│        （管理工具 / 开发工作区）                     │
│                                                     │
│  - 配置 OpenClaw 系统                               │
│  - 开发新 Agent / Skills                            │
│  - 执行优化计划                                      │
│  - 管理团队组合                                      │
│  - 维护决策算法                                      │
└──────────────────────┬──────────────────────────────┘
                       │ 操作/管理（单向）
                       ▼
┌─────────────────────────────────────────────────────┐
│                   OpenClaw 系统                      │
│                  （~/.claw/）                        │
│                                                     │
│  - 22 个 Agent（Queen + 6 妃子 + 7 尚座 + 8 司师）   │
│  - 六局一司架构                                      │
│  - 多渠道接入（飞书、Telegram）                      │
│  - Skills 扩展能力                                   │
└─────────────────────────────────────────────────────┘
```

**关键点**：
- **Claw 操作 OpenClaw**，OpenClaw 不知道 Claw 的存在
- 两套系统**互不引用**，是管理与被管理关系
- 你在 Claw 里操作，实际改的是 `~/.claw/` 下的文件

### 1.3 目录路径说明

| 项目 | 路径 | 角色 | 规则文件 |
|------|------|------|----------|
| Claw | `/Users/linwei/CodeBuddy/Claw/` | 管理工具（你现在的位置） | `CODEBUDDY.md`（本文件） |
| OpenClaw | `~/.claw/` | 被管理的系统 | `CLAUDE.md`（给 Claude Code 用） |

> **注**：`~/.openclaw` 是指向 `~/.claw` 的符号链接，两者等同。项目最初叫 "Claw"，后改名 "OpenClaw"，目录名沿用旧名。

## 三、操作规范

### 3.1 Skill 开发规范

**目录结构**：
```
skill-name/               # kebab-case ONLY
├── SKILL.md              # 必须大写
├── references/           # 参考文档
├── scripts/              # 脚本
└── assets/               # 资源文件
```

**YAML Frontmatter**：
```yaml
---
description: 技能描述。触发词：关键词1、关键词2
---
```

### 3.2 命名规范

- 司级、典级名称必须为 **3个字**
- 所有职级名称在全局范围内 **不能重复**

### 3.3 操作前检查清单

| 操作 | 必做检查 |
|------|----------|
| 修改配置 | 备份原文件 |

| 新增 Skill | 验证 SKILL.md 格式 |
| 部署变更 | 读取 `PROGRESS.md` 了解当前状态 |

## 四、输出排版标准

**继承自** `~/.claw/OUTPUT-STANDARD.md`，核心规则：

### 4.1 标题层级

| 层级 | 语法 | 用途 |
|------|------|------|
| 一级 `#` | 仅文档标题 | 每篇输出只用 1 次 |
| 二级 `##` | 主要章节 | 划分大板块 |
| 三级 `###` | 子章节 | 章节内细分 |

### 4.2 表格优先

以下信息**必须用表格**：
- 对比数据（多方案、多指标）
- 参数配置（键值对）
- 状态汇总（进度、结果）
- 角色职责（人/Agent + 职责）

### 4.3 长度控制

| 场景 | 建议长度 |
|------|----------|
| 快速问答 | ≤ 5 行 |
| 执行结果 | ≤ 20 行 |
| 分析报告 | ≤ 50 行 |

### 4.4 排版禁忌

| 禁止 | 替代方案 |
|------|----------|
| 大段无标题文字 | 拆分为 `##` 小节 |
| 连续 2 个以上空行 | 最多空 1 行 |
| 纯文字描述数据 | 用表格 |
| emoji 超过 3 个 | 仅标题用 1 个 |

### 6.1 开始工作前

2. 读取 `~/.claw/CLAUDE.md` 了解架构和规范

- 原因
- 影响
