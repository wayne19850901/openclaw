## 一、系统架构

```
┌─────────────────────────────────────────────────────────────┐

                       │ 确认晋升
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                    Queen 👑                                   │

└──────────────────────┬──────────────────────────────────────┘
                       │
        ┌──────────────┼──────────────┐
        ▼              ▼              ▼

│   全局记忆    │ │   局级记忆    │ │   妃子记忆    │
│ ~/self-      │ │ ~/.claw/     │ │ ~/.claw/     │
│ improving/   │ │ agents/<局>/ │ │ agents/<妃>/ │
│              │ │ .learnings/  │ │ .learnings/  │
└──────────────┘ └──────────────┘ └──────────────┘
        │              │              │
        └──────────────┼──────────────┘
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                  OpenClaw 规则文件                            │
│  ~/.claw/CLAUDE.md     — 项目事实和约定                       │
│  ~/.claw/OUTPUT-STANDARD.md — 输出排版标准                    │
│  ~/.claw/PROGRESS.md   — 项目进度                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、记忆层级

| 层级 | 位置 | 大小限制 | 行为 |
|------|------|----------|------|
| HOT | `~/self-improving/memory.md` | ≤100 行 | 始终加载 |
| WARM | `~/.claw/agents/<局>/.learnings/` | ≤200 行/文件 | 按需加载 |
| COLD | `~/self-improving/archive/` | 无限制 | 显式查询 |

---

## 三、晋升机制

```
局级学习 (.learnings/)
    ↓ 重复3次 + 适用范围扩大
全局学习 (~/self-improving/)
    ↓ 确认为系统规则
OpenClaw规则 (~/.claw/)
```

### 3.1 晋升条件

| 源 → 目标 | 条件 | 审查者 |
|-----------|------|--------|
| `.learnings/` → `~/self-improving/` | 重复3次 + 跨局适用 | Queen |
| `~/self-improving/` → `~/.claw/CLAUDE.md` | 确认为系统规则 | Queen + 威帝 |
| `~/self-improving/` → `~/.claw/OUTPUT-STANDARD.md` | 排版相关规则 | Queen + 威帝 |

### 3.2 降级条件

| 条件 | 操作 |
|------|------|
| 30天未使用 | 降级到 WARM |
| 90天未使用 | 归档到 COLD |
| 规则冲突 | 标记待确认 |

## 四、学习触发器

### 4.1 通用触发器

| 信号 | 记录到 | 类别 |
|------|--------|------|
| "不对，应该是..." | `LEARNINGS.md` | correction |
| "你写错了..." | `LEARNINGS.md` | correction |

每个局有自己的领域特定触发器，定义在各自的 `SELF-IMPROVING.md` 中。

## 五、心跳维护

### 5.1 频率

| 层级 | 频率 | 执行者 |
|------|------|--------|
| 局级 | 每周（周日） | 各局尚座 |
| 妃子级 | 每两周 | 各妃子 |
| 全局 | 每月（1日） | Queen |
| 晋升检查 | 每季度 | Queen + 御鉴司 |

### 5.2 心跳任务

1. 检查待处理学习
2. 检查重复学习
3. 检查晋升候选
4. 更新心跳状态

## 六、使用指南

### 6.1 记录学习

当发现需要记录的学习时：

1. **确定类别**：correction / best_practice / knowledge_gap / error / feature_request
2. **确定层级**：局级 / 妃子级 / 全局
3. **记录到对应文件**：
   - 局级：`~/.claw/agents/<局名>/.learnings/LEARNINGS.md`
   - 全局：`~/self-improving/memory.md`
4. **标记元数据**：Pattern-Key, Recurrence-Count, Cross-Bureau 等

### 6.2 查询学习

```bash
# 查看所有待处理学习
grep -rh "Status\*\*: pending" ~/.claw/agents/*/.learnings/*.md

# 查看晋升候选
grep -rh "Recurrence-Count: [3-9]" ~/.claw/agents/*/.learnings/*.md

# 查看跨局学习
grep -rh "Cross-Bureau: true" ~/.claw/agents/*/.learnings/*.md
```

### 6.3 晋升学习

当学习满足晋升条件时：

1. **局级 → 全局**：
   - 尚座提交晋升申请
   - Queen 审查
   - 御鉴司合规审查
   - 通过后复制到 `~/self-improving/memory.md`

2. **全局 → OpenClaw 规则**：
   - Queen 提议晋升
   - 御鉴司审查
   - 威帝确认
   - 更新对应规则文件
