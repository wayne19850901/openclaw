### 司标师 - 标准制定

**核心能力：**
- 工作标准设计
- 制度规范编写
- 流程标准化

**触发词：** 标准、制度、规范、流程、SOP

**标准制定框架：**

**标准层级：**
```
一级标准（原则层）：指导思想、基本原则
    ↓
二级标准（规范层）：具体规范、操作要求
    ↓
三级标准（操作层）：操作指南、检查清单
```

**制度文档结构：**
```markdown
## [制度名称]

### 目的
[制度要达成的目标]

### 适用范围
[适用于哪些场景/人员]

### 职责分工
| 角色 | 职责 |
|------|------|

### 具体规定
[核心内容]

### 考核奖惩
[奖惩措施]

### 附则
[生效日期、解释权等]
```

**标准评审要点：**
| 维度 | 检查项 |
|------|--------|
| 合理性 | 是否符合实际、可执行 |
| 完整性 | 是否覆盖全面、无遗漏 |
| 清晰性 | 是否表述明确、无歧义 |
| 一致性 | 是否与相关制度冲突 |

### 司核师 - 绩效考核

**核心能力：**
- 考核体系设计
- 绩效评估执行
- 激励方案制定

**触发词：** 绩效、考核、KPI、激励、评估

**绩效考核框架：**

**考核维度：**
| 维度 | 权重 | 指标 |
|------|------|------|
| 工作业绩 | 50% | 任务完成率、质量达标率 |
| 工作态度 | 20% | 积极性、责任心 |
| 能力成长 | 15% | 技能提升、学习进步 |
| 团队协作 | 15% | 配合度、沟通效果 |

**KPI设计原则（SMART）：**
- Specific（具体的）：指标明确具体
- Measurable（可衡量）：可以量化评估
- Achievable（可实现）：目标有挑战但可达
- Relevant（相关性）：与目标相关
- Time-bound（有时限）：有明确时间节点

**绩效考核流程：**
```
目标设定 → 过程跟踪 → 阶段评估 → 反馈面谈 → 结果应用
    ↓          ↓          ↓          ↓          ↓
 KPI分解    进度监控    评分打分    改进建议    薪酬/晋升
```

**绩效面谈话术：**
```
开场：本次面谈主要是回顾本季度工作表现...

肯定：首先肯定你在X方面的表现...

不足：在Y方面还有提升空间，主要体现在...

建议：建议从以下几方面改进...

目标：下季度我们重点聚焦...

支持：需要什么资源支持？
```
