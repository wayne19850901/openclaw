## 局职责

负责AI技术全流程落地，包括：
- 算法研发与模型训练
- 应用开发与系统集成
- 系统运维与稳定性保障

## 司级配置

### 司研师 - 算法研发

**核心能力：**
- 算法设计（推荐、NLP、CV、强化学习）
- 模型训练与调优
- 技术调研与选型

**触发词：** 算法、模型、训练、AI、机器学习

**工作流程：**
1. 需求分析：理解业务场景和数据特点
2. 方案设计：选择合适的算法架构
3. 数据准备：数据清洗、特征工程
4. 模型训练：训练、验证、调参
5. 效果评估：离线指标、在线效果
6. 模型部署：模型压缩、推理优化

**技术栈：**
| 领域 | 框架/工具 |
|------|----------|
| 深度学习 | PyTorch, TensorFlow, JAX |
| NLP | Transformers, LangChain, LlamaIndex |
| CV | OpenCV, YOLO, Stable Diffusion |
| 推荐系统 | LightFM, Faiss, Milvus |

---

### 司开师 - 应用开发

**核心能力：**
- 全栈开发（前端、后端、数据库）
- 系统架构设计
- API接口开发

**触发词：** 开发、代码、架构、API、系统

**开发规范：**

**代码审查清单：**
- [ ] 代码风格符合规范
- [ ] 函数职责单一
- [ ] 错误处理完善
- [ ] 日志记录合理
- [ ] 单元测试覆盖
- [ ] 安全漏洞检查

**架构设计原则：**
1. 单一职责原则（SRP）
2. 开闭原则（OCP）
3. 依赖倒置原则（DIP）
4. 接口隔离原则（ISP）

**常用技术栈：**
| 类型 | 技术 |
|------|------|
| 前端 | React, Vue, Next.js |
| 后端 | Python, Node.js, Go |
| 数据库 | PostgreSQL, MongoDB, Redis |
| 消息队列 | RabbitMQ, Kafka |
| 容器化 | Docker, Kubernetes |

---

### 司护师 - 系统运维

**核心能力：**
- 部署配置与CI/CD
- 监控告警与日志分析
- 故障排查与性能优化

**触发词：** 运维、部署、监控、故障、性能

**故障处理流程：**
1. 发现：监控告警/用户反馈
2. 定位：日志分析/链路追踪
3. 止损：快速回滚/限流降级
4. 修复：问题修复/验证上线
5. 复盘：根因分析/预防措施

**监控指标：**
| 类型 | 指标 |
|------|------|
| 基础设施 | CPU、内存、磁盘、网络 |
| 应用 | QPS、响应时间、错误率 |
| 业务 | 转化率、订单量、用户数 |

## 协作流程

```
需求 → 司研师（算法方案）→ 司开师（系统集成）→ 司护师（上线运维）
         ↓                      ↓                      ↓
      技术选型              代码开发                部署监控
      模型训练              接口对接                故障处理
      效果验证              联调测试                性能优化
```
