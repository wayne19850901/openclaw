## 阶段1：任务接收与评估

### 步骤1：接收指令

- 来源：Queen指令 / 威帝直接指令 / 跨局协作请求
- 记录任务来源、时间、优先级
- [CHECKPOINT-1]：指令已记录

### 步骤2：任务归属判断

```
评估任务类型
    ├── 金融投资类 → 司投师
    ├── 信贷贷款类 → 司贷师
    ├── 客户管理类 → 司客师
    ├── 跨局协调类 → 自身执行
    ├── 简单问答类 → 自身执行
    └── 超出能力圈 → 上报Queen
```

- [CHECKPOINT-2]：归属已判断

### 步骤3：能力圈检查

- 任务是否在金融贷款领域内？
- 是否需要专业工具（DCF、风控模型）？
- 是否涉及跨局协作？

- [CHECKPOINT-3]：能力圈已确认

---

## 阶段2：任务分发与协调

### 步骤4：分发任务

- 明确交付标准（护城河评估、安全边际、长期价值）
- 设定截止时间
- 指定验收标准
- [CHECKPOINT-4]：任务已分发

### 步骤5：跨局协调（如需）

- 识别协作局（尚工局/尚宣局/尚机局等）
- 联系对应妃子
- 明确协作接口和交付物
- [CHECKPOINT-5]：协调已建立

### 步骤6：进度跟踪

- 定期检查任务进度
- 识别阻塞项
- 及时调整资源
- [CHECKPOINT-6]：进度已跟踪

---

## 阶段3：质量门控验收

### 步骤7：护城河评估检查

- 品牌护城河是否清晰
- 成本优势是否可持续
- 网络效应是否显著
- 转换成本是否充分
- [CHECKPOINT-7]：护城河已评估

### 步骤8：安全边际检查

- 风险收益比是否≥3:1
- 内在价值-价格是否≥25%
- 下行保护是否充分
- [CHECKPOINT-8]：安全边际已检查

### 步骤9：长期价值验证

- LTV是否≥3倍CAC
- 客户留存率是否≥80%
- 复购率是否≥30%
- [CHECKPOINT-9]：长期价值已验证

### 步骤10：验收决定

```
验收结果
    ├── 全部通过 → 汇报Queen
    ├── 部分不通过 → 退回重做（附修改意见）
    └── 重大缺陷 → 退回重做 + 上报Queen
```

- [CHECKPOINT-10]：验收已完成

---

## 阶段4：汇报反馈

### 步骤11：成果汇报

- 向Queen汇报任务结果
- 包含：交付物、验收结果、关键发现
- 标注风险项和后续建议
- [CHECKPOINT-11]：已汇报Queen

### 步骤12：知识沉淀

- 记录成功案例到参考库
- 更新威帝偏好数据
- 优化任务分发规则
- [CHECKPOINT-12]：知识已沉淀
