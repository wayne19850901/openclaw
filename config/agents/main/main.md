## 妃子召唤路由

### 群聊智能路由 v3.0

**多维打分 + 上下文感知 + 偏好学习 + 召回机制**

#### 核心机制

| 机制 | 说明 |
|------|------|
| **多维打分** | 领域40% + 场景30% + 类型20% + 偏好10% |
| **上下文感知** | 记录话题、激活妃子、历史关联 |
| **偏好学习** | 记录用户偏好，持续优化 |
| **召回纠正** | 用户可纠正路由错误 |

#### 路由流程

```
消息 → 紧急判断 → 上下文检查 → 多维打分 → 偏好加分 → 召唤妃子 → 更新上下文
```

#### 多妃子协同

| 情况 | 处理 |
|------|------|
| 最高分 > 次高分 20分+ | 只召唤1个妃子 |
| 差距 < 20分 | 召唤前2-3名协同 |
| 所有分数 < 50分 | Queen 处理或询问 |

#### 用户命令

| 命令 | 作用 |
|------|------|
| /retry | 重新判断路由 |
| /clear | 清空上下文 |
| /status | 查看当前话题 |
| "不对，让XX来" | 纠正路由 |

#### 配置文件

- `memory/routing-config.json` - 路由配置
- `memory/conversation-context.json` - 上下文存储
- `memory/consort-preferences.json` - 用户偏好

详细路由规则：`references/consort-routing.md`
