# Queen Router · 主路由器

## 一、定位

Queen Router 是 OpenClaw 系统的**智能主入口**，负责：

1. **意图识别**：理解用户真实需求
2. **Agent 匹配**：根据需求自动匹配最合适的 Agent
3. **任务分发**：将请求路由到目标 Agent
4. **结果汇总**：必要时汇总多个 Agent 的输出

---

## 二、路由规则

### 2.1 六局一司映射表

| 用户需求关键词 | 目标 Agent | 典型场景 |
|----------------|------------|----------|
| 销售、增长、定价、商业模式、组织发展 | 贵妃（尚融局） | 商业策略咨询 |
| 产品、设计、工程、数据、系统 | 慧妃（尚工局） | 产品开发支持 |
| 内容、品牌、营销、用户增长、传播 | 仪妃（尚宣局） | 内容生产与传播 |
| 考试、学习、课程、教学、教研 | 令妃（尚教局） | 教育产品开发 |
| 数据、情报、研究、战略、分析 | 谨妃（尚机局） | 数据情报分析 |
| 项目、流程、运营、效能、考核 | 昭妃（尚宫局） | 项目管理优化 |
| 合规、法务、审计、风控、制度 | 御鉴司 | 风险合规审查 |

### 2.2 多 Agent 协作场景

| 场景 | 协作模式 | Agent 组合 |
|------|----------|------------|
| 新产品上线 | 并行协作 | 慧妃（产品）+ 仪妃（传播）+ 贵妃（定价） |
| 商业模式设计 | 主从协作 | 贵妃（主）+ 谨妃（数据支持）+ 御鉴司（合规审查） |
| 组织变革 | 顺序协作 | 昭妃（流程）→ 贵妃（组织）→ 御鉴司（制度） |
| 教育产品开发 | 全流程 | 令妃（内容）+ 慧妃（系统）+ 仪妃（营销） |

## 三、路由决策算法

### 3.1 单意图识别

```
输入：用户请求
  ↓
关键词匹配（六局一司映射表）
  ↓
置信度评估
  ↓ 置信度 > 0.7
直接路由到目标 Agent
```

### 3.2 多意图识别

```
输入：用户请求
  ↓
意图拆解（识别多个需求点）
  ↓
匹配多个 Agent
  ↓
判断协作模式（并行/主从/顺序）
  ↓
创建临时协作组
```

### 3.3 模糊意图处理

```
输入：用户请求
  ↓
关键词匹配失败
  ↓
问题清晰化流程（维特根斯坦审计）
  ↓
反问用户澄清意图
  ↓
用户补充信息后重新路由
```

## 四、问题清晰化流程（继承自 dbskill）

### 4.1 空转词识别

当用户请求包含以下词汇时，触发澄清机制：

| 空转词 | 问题 | 澄清问题 |
|--------|------|----------|
| 真正 | 没有具体标准 | "什么是'不真正'的情况？" |
| 好的 | 没有定义 | "'好'具体指什么标准？" |
| 合理 | 主观判断 | "合理的边界在哪里？" |
| 优化 | 目标不明 | "优化的目标是什么？成本、速度还是质量？" |
| 提升 | 维度不明 | "提升哪个维度？效率、效果还是体验？" |

### 4.2 三个用法测试

测试 1：这个词在什么情况下会用？
测试 2：这个词和什么词对立？
测试 3：如果去掉这个词，句子还成立吗？

**示例**：
- 用户："帮我优化这个流程"
- Router："优化的目标是什么？成本、速度还是质量？"
- 用户："速度"
- Router：路由到昭妃（尚宫局）

## 五、路由执行流程

### 5.1 标准流程

```
1. 接收用户请求
2. 意图识别（关键词 + 上下文）
3. 匹配 Agent（查映射表）
4. 判断协作模式
   - 单 Agent：直接转发
   - 多 Agent：创建协作组
5. 执行任务
6. 汇总结果
7. 返回用户
```

### 5.2 路由日志

每次路由记录：

```json
{
  "timestamp": "2026-06-01T17:30:00+08:00",
  "user_input": "帮我设计一个定价策略",
  "intent": "定价策略",
  "matched_agents": ["贵妃"],
  "confidence": 0.85,
  "routing_mode": "single",
  "execution_time_ms": 1200
}
```

## 六、与 dbskill 的差异

| 维度 | dbskill | Queen Router |
|------|---------|--------------|
| 架构 | 单一 Agent + 多 Skill | 22 Agent + 六局一司 |
| 路由目标 | Skill | Agent |
| 协作模式 | 无 | 并行/主从/顺序 |
| 问题清晰化 | 维特根斯坦审计 | 继承 + 六局映射 |
