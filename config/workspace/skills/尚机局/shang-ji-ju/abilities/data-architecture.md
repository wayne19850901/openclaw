# 数据架构

## 概述

建立集团统一的情报数据库，确保数据"存得进、找得快"。

## 核心职责

- 情报数据库设计
- 数据清洗流程
- 数据质量管理
- 检索性能优化

## 数据架构

```
┌─────────────────────────────────────────────────────────┐
│ 数据接入层                                              │
│    ├── 爬虫数据                                         │
│    ├── API数据                                          │
│    └── 手工录入                                         │
├─────────────────────────────────────────────────────────┤
│ 数据处理层                                              │
│    ├── 清洗（去重、补全、标准化）                        │
│    ├── 标注（分类、打标）                               │
│    └── 向量化（Embedding）                              │
├─────────────────────────────────────────────────────────┤
│ 数据存储层                                              │
│    ├── Elasticsearch（全文检索）                        │
│    ├── MongoDB（文档存储）                              │
│    ├── Pinecone（向量检索）                             │
│    └── PostgreSQL（结构化数据）                         │
├─────────────────────────────────────────────────────────┤
│ 数据服务层                                              │
│    ├── 搜索API                                          │
│    ├── 分析API                                          │
│    └── 导出API                                          │
└─────────────────────────────────────────────────────────┘
```

## 数据模型

### 情报文档结构

```json
{
  "id": "unique_id",
  "title": "情报标题",
  "content": "正文内容",
  "source": "来源",
  "category": "分类",
  "tags": ["标签1", "标签2"],
  "sentiment": "positive/negative/neutral",
  "importance": 1-5,
  "publish_time": "2026-04-06T00:00:00Z",
  "collect_time": "2026-04-06T00:00:00Z",
  "metadata": {}
}
```

## 清洗规则

| 步骤 | 规则 |
|------|------|
| 去重 | 基于标题+内容的相似度去重 |
| 补全 | 自动补充时间、来源等字段 |
| 标准化 | 统一格式、编码、时区 |
| 质检 | 检查必填字段、格式正确性 |

## 性能指标

| 指标 | 目标 |
|------|------|
| 写入吞吐 | > 10000条/秒 |
| 检索延迟 | < 100ms |
| 存储容量 | PB级 |
