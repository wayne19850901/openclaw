# 司试师假设测试工作流

> **执行者**: 司试师（科学方法论）
> **用途**: 系统化假设测试流程
> **约束**: 必须完整执行，不得跳过任何步骤
> **强制读完**: 本文件包含 `[EXECUTION_COMPLETE]` 标记，AI必须读到该标记才算完成

---

## 工作流概述

本工作流基于科学方法论，采用 **假设提出 + 实验设计 + 数据收集 + 统计分析 + 结论验证** 五大核心方法论。

**执行原则**：
1. **强制读完机制**：AI必须执行完整流程，不得跳过任何步骤
2. **假设可证伪性**：假设必须可被证伪
3. **数据样本量**：样本量≥30，确保统计显著性
4. **统计显著性**：p值<0.05才可接受假设

---

## Phase 1: 假设提出（强制执行）

### 1.1 明确假设目标

**任务**: 明确假设测试的目标和背景。

**执行步骤**：
1. 明确假设目标：
   - 假设类型（投资/产品/运营）
   - 假设背景
   - 假设重要性

**输出格式**：
```
【假设目标】
- 假设类型：[类型]
- 假设背景：[背景描述]
- 假设重要性：[重要性描述]
```

**强制检查点**：必须明确假设目标，缺失不得进入下一阶段。

---

### 1.2 提出可证伪假设

**任务**: 提出清晰、可验证、可证伪的假设。

**执行步骤**：
1. 提出假设（H0和H1）：
   - H0（零假设）：[假设描述]
   - H1（备择假设）：[假设描述]
2. 确保假设可证伪：
   - 假设清晰度评分≥7分
   - 假设可验证性评分≥7分
   - 假设可证伪性评分≥7分

**输出格式**：
```
【假设提出】
- H0（零假设）：[假设描述]
- H1（备择假设）：[假设描述]
- 假设清晰度评分：[评分]/10
- 假设可验证性评分：[评分]/10
- 假设可证伪性评分：[评分]/10
```

**强制检查点**：
- 假设评分均≥7分：进入下一阶段
- 假设评分<7分：**立即修改假设后继续**

---

## Phase 2: 实验设计（强制执行）

### 2.1 设计实验方案

**任务**: 设计科学实验方案，验证假设。

**执行步骤**：
1. 设计实验方案：
   - 实验类型（A/B测试/对照实验/观察实验）
   - 实验变量（自变量/因变量/控制变量）
   - 实验样本量（≥30）
   - 实验随机分组

**输出格式**：
```
【实验设计】
- 实验类型：[类型]
- 自变量：[变量描述]
- 因变量：[变量描述]
- 控制变量：[变量描述]
- 样本量：[数量]（≥30）
- 随机分组：[分组方案]
```

**强制检查点**：
- 样本量≥30：进入下一阶段
- 样本量<30：**增加样本量后继续**

---

### 2.2 实验设计评分

**任务**: 对实验设计进行评分。

**评分标准**（0-10分）：
- **9-10分**：实验设计科学，变量控制严格，样本量充足
- **7-8分**：实验设计良好，变量控制较好，样本量充足
- **5-6分**：实验设计一般，变量控制一般，样本量充足
- **3-4分**：实验设计较差，变量控制不足，样本量不足
- **0-2分**：实验设计失败，无法验证假设

**输出格式**：
```
【实验设计评分】
- 变量控制评分：[评分]/10
- 样本量评分：[评分]/10
- 随机分组评分：[评分]/10
- 实验设计总分：[总分]/10
```

**强制检查点**：
- 实验设计总分≥6分：进入下一阶段
- 实验设计总分<6分：**立即终止，输出"不测试"决策**

---

## Phase 3: 数据收集（强制执行）

### 3.1 执行数据收集

**任务**: 执行实验，收集数据。

**执行步骤**：
1. 执行实验
2. 收集数据：
   - 数据量≥30个样本
   - 数据质量评分≥7分
   - 数据完整性≥90%

**输出格式**：
```
【数据收集】
- 数据量：[数量]个样本
- 数据质量评分：[评分]/10
- 数据完整性：[完整性]%
- 数据来源：[来源描述]
```

**强制检查点**：
- 数据量≥30且质量≥7分：进入下一阶段
- 数据量<30或质量<7分：**补充数据后继续**

---

## Phase 4: 统计分析（强制执行）

### 4.1 选择统计方法

**任务**: 选择合适的统计方法分析数据。

**执行步骤**：
1. 选择统计方法：
   - t检验（两组对比）
   - 卡方检验（分类数据）
   - ANOVA（多组对比）
   - 回归分析（相关性分析）

**输出格式**：
```
【统计方法选择】
- 统计方法：[方法名称]
- 方法适用性：[适用性描述]
- 方法准确性：[准确性描述]
```

**强制检查点**：必须选择合适的统计方法，缺失不得进入下一阶段。

---

### 4.2 执行统计分析

**任务**: 执行统计分析，计算p值。

**执行步骤**：
1. 执行统计分析
2. 计算p值
3. 计算效应量

**输出格式**：
```
【统计分析结果】
- 统计方法：[方法名称]
- p值：[p值]
- 统计显著性：p值<0.05？[是/否]
- 效应量：[效应量]
- 效应量大小：[大/中/小]
```

**强制检查点**：
- p值<0.05：进入下一阶段
- p值≥0.05：**立即终止，输出"拒绝假设"决策**

---

## Phase 5: 结论验证（强制执行）

### 5.1 结论一致性验证

**任务**: 验证结论与假设的一致性。

**执行步骤**：
1. 对比结论与假设
2. 验证结论一致性
3. 验证结论可靠性

**输出格式**：
```
【结论验证】
- H0假设：[假设描述]
- 结论：[结论描述]
- 一致性：[是/否]
- 可靠性评分：[评分]/10
```

**强制检查点**：
- 一致性=是且可靠性≥7分：进入下一阶段
- 一致性=否或可靠性<7分：**立即终止，输出"修改假设"决策**

---

### 5.2 结论推广性验证

**任务**: 验证结论的可推广性。

**执行步骤**：
1. 验证结论可重复性
2. 验证结论推广范围
3. 验证结论局限性

**输出格式**：
```
【结论推广性】
- 可重复性：[可重复性描述]
- 推广范围：[推广范围描述]
- 局限性：[局限性描述]
- 推广性评分：[评分]/10
```

**强制检查点**：必须验证结论推广性，缺失不得进入下一阶段。

---

## Phase 6: 最终决策（强制执行）

### 6.1 假设测试评分汇总

**任务**: 汇总所有评分，生成最终决策。

**评分权重**：
- 假设提出：20%
- 实验设计：20%
- 数据收集：20%
- 统计分析：20%
- 结论验证：20%

**执行步骤**：
1. 计算综合评分（加权平均）
2. 给出最终决策（ACCEPT/MODIFY/REJECT）
3. 给出信心水平（高/中/低）

**输出格式**：
```
【综合评分汇总】
评分项          得分    权重    加权得分
假设提出        [得分]  20%     [加权得分]
实验设计        [得分]  20%     [加权得分]
数据收集        [得分]  20%     [加权得分]
统计分析        [得分]  20%     [加权得分]
结论验证        [得分]  20%     [加权得分]
------------------------------------------------
综合评分        [综合评分]（满分10分）

【最终决策】
- 决策：ACCEPT / MODIFY / REJECT
- 信心水平：高 / 中 / 低
- p值：[p值]
- 效应量：[效应量]
```

**强制检查点**：必须给出明确的 ACCEPT/MODIFY/REJECT 决策。

---

### 6.2 假设测试建议

**任务**: 给出具体假设测试建议。

**输出格式**：
```
【假设测试建议】
- 假设状态：[接受/修改/拒绝]
- 下一步行动：[行动描述]
- 重复实验建议：[建议描述]
- 推广应用建议：[建议描述]
```

**强制检查点**：必须给出假设状态和下一步行动建议，缺失不得完成工作流。
