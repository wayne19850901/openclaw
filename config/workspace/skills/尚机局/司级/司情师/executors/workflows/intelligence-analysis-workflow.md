# 司情师情报分析工作流

> **执行者**: 司情师（Stanley Druckenmiller哲学）
> **用途**: 系统化情报分析流程
> **约束**: 必须完整执行，不得跳过任何步骤
> **强制读完**: 本文件包含 `[EXECUTION_COMPLETE]` 标记，AI必须读到该标记才算完成

---

## 工作流概述

本工作流基于 Stanley Druckenmiller 的宏观投资哲学，采用 **自上而下分析 + 趋势跟踪 + 盈亏比管理** 三大核心方法论。

**执行原则**：
1. **强制读完机制**：AI必须执行完整流程，不得跳过任何步骤
2. **数据源唯一性**：所有金融数据必须使用 `neodata-financial-search` skill
3. **量化指标验证**：每个决策必须有量化指标支撑（盈亏比≥3:1、趋势强度评分）
4. **宏观优先原则**：必须先完成宏观分析，再进行行业和公司分析

---

## Phase 1: 宏观环境分析（强制执行）

### 1.1 全球宏观经济指标

**任务**: 获取全球宏观经济指标。

**执行步骤**：
1. 调用 `neodata-financial-search` skill
2. 搜索以下宏观指标：
   - 美国GDP增长率
   - 中国GDP增长率
   - 美联储利率
   - 中国央行利率
   - 美元指数
   - 人民币汇率
   - CPI（美国/中国）
   - PMI（美国/中国）

**输出格式**：
```
【全球宏观经济指标】
指标                美国          中国
GDP增长率(%)        [数据]       [数据]
央行利率(%)         [数据]       [数据]
CPI(%)              [数据]       [数据]
PMI                 [数据]       [数据]
汇率                美元指数：[数据]  人民币汇率：[数据]
```

**强制检查点**：必须获取至少6项宏观指标，缺失不得进入下一阶段。

---

### 1.2 宏观趋势判断

**任务**: 判断当前宏观经济趋势。

**趋势判断标准**：
- **扩张期**：GDP增长 > 3%，PMI > 50，利率上升
- **衰退期**：GDP增长 < 2%，PMI < 50，利率下降
- **复苏期**：GDP增长由负转正，PMI回升
- **滞胀期**：GDP增长放缓，CPI上升

**执行步骤**：
1. 分析GDP增长趋势
2. 分析PMI趋势
3. 分析利率趋势
4. 综合判断宏观周期

**输出格式**：
```
【宏观趋势判断】
- GDP趋势：[趋势分析]
- PMI趋势：[趋势分析]
- 利率趋势：[趋势分析]
- 宏观周期：扩张期 / 衰退期 / 复苏期 / 滞胀期
- 信心水平：高 / 中 / 低
```

**强制检查点**：必须给出明确的宏观周期判断，缺失不得进入下一阶段。

---

### 1.3 宏观风险因素

**任务**: 识别宏观风险因素。

**执行步骤**：
1. 分析地缘政治风险
2. 分析货币政策风险
3. 分析贸易摩擦风险
4. 分析系统性金融风险

**输出格式**：
```
【宏观风险因素】
1. [风险因素1] - 影响程度：高/中/低
2. [风险因素2] - 影响程度：高/中/低
3. [风险因素3] - 影响程度：高/中/低
```

**强制检查点**：必须识别至少3项宏观风险因素，缺失不得进入下一阶段。

---

## Phase 2: 行业分析（强制执行）

### 2.1 目标行业选择

**任务**: 基于宏观分析，选择目标行业。

**行业选择标准**：
- **顺周期行业**：宏观扩张期，选择科技、消费、金融
- **逆周期行业**：宏观衰退期，选择医疗、公用事业、必需消费品
- **成长行业**：宏观复苏期，选择新能源、AI、半导体

**执行步骤**：
1. 基于宏观周期，筛选3-5个目标行业
2. 分析行业增长潜力
3. 分析行业竞争格局

**输出格式**：
```
【目标行业选择】
行业        选择理由                      增长潜力(高/中/低)
[行业1]     [理由]                       [潜力]
[行业2]     [理由]                       [潜力]
[行业3]     [理由]                       [潜力]
```

**强制检查点**：必须选择至少3个目标行业，缺失不得进入下一阶段。

---

### 2.2 行业竞争格局分析

**任务**: 分析目标行业的竞争格局。

**执行步骤**：
1. 调用 `neodata-financial-search` skill
2. 搜索行业规模和增长率
3. 搜索主要竞争对手市场份额
4. 分析行业集中度

**输出格式**：
```
【行业竞争格局】
- 行业规模：[规模]亿元
- 行业增长率：[增长率]%
- 主要竞争对手：
  1. [竞争对手1] - 市场份额：[份额]%
  2. [竞争对手2] - 市场份额：[份额]%
  3. [竞争对手3] - 市场份额：[份额]%
- 行业集中度（CR3）：[集中度]%
```

**强制检查点**：必须获取行业规模和至少3家竞争对手信息，缺失不得进入下一阶段。

---

### 2.3 行业趋势强度评分

**任务**: 对目标行业进行趋势强度评分。

**趋势强度评分标准**（0-10分）：
- **行业增长率**：0-3分（增长<5%=0分，5-10%=1分，10-20%=2分，>20%=3分）
- **政策支持度**：0-3分（不支持=0分，中性=1分，支持=2分，强力支持=3分）
- **技术成熟度**：0-2分（早期=0分，成长期=1分，成熟期=2分）
- **资本关注度**：0-2分（低=0分，中=1分，高=2分）

**执行步骤**：
1. 评估行业增长率评分
2. 评估政策支持度评分
3. 评估技术成熟度评分
4. 评估资本关注度评分
5. 计算综合趋势强度评分

**输出格式**：
```
【行业趋势强度评分】
评分项          得分(0-10)    评分依据
行业增长率      [得分]        [依据]
政策支持度      [得分]        [依据]
技术成熟度      [得分]        [依据]
资本关注度      [得分]        [依据]
------------------------------------------------
综合趋势强度    [综合评分]    （满分10分）
```

**强制检查点**：
- 综合趋势强度 ≥ 6分：进入下一阶段
- 综合趋势强度 < 6分：**立即终止分析，输出"不投资"决策**

## Phase 3: 公司分析（强制执行）

### 3.1 目标公司筛选

**任务**: 从目标行业中筛选目标公司。

**筛选标准**：
- 市值规模适中（50-500亿）
- 行业地位领先（前3名）
- 财务状况健康（ROE>15%，现金流为正）

**执行步骤**：
1. 调用 `neodata-financial-search` skill
2. 搜索目标行业的龙头企业
3. 筛选符合标准的公司

**输出格式**：
```
【目标公司筛选】
公司名称    市值(亿)    行业排名    ROE(%)    现金流(亿)
[公司1]     [数据]      [排名]     [数据]    [数据]
[公司2]     [数据]      [排名]     [数据]    [数据]
[公司3]     [数据]      [排名]     [数据]    [数据]
```

**强制检查点**：必须筛选至少3家目标公司，缺失不得进入下一阶段。


**强制检查点**：必须获取过去5年的完整财务数据，缺失不得进入下一阶段。

### 3.3 公司竞争优势分析

**任务**: 分析目标公司的竞争优势。

**竞争优势要素**：
- 技术壁垒
- 品牌优势
- 规模效应
- 渠道优势
- 成本优势

**执行步骤**：
1. 分析每个竞争优势要素的证据
2. 给出评分（0-10分）
3. 计算综合竞争优势评分

**输出格式**：
```

```

**强制检查点**：必须完成竞争优势评分，缺失不得进入下一阶段。

## Phase 4: 趋势跟踪分析（强制执行）

### 4.1 价格趋势分析

**任务**: 分析目标公司的价格趋势。

**趋势判断标准**：
- **上升趋势**：股价 > MA50 > MA200
- **下降趋势**：股价 < MA50 < MA200
- **震荡趋势**：MA50与MA200交叉

**执行步骤**：
1. 调用 `neodata-financial-search` skill
2. 搜索过去1年的股价数据
3. 计算MA50和MA200
4. 判断趋势方向

**输出格式**：
```
【价格趋势分析】
- 当前股价：[股价]元
- MA50：[MA50]元
- MA200：[MA200]元
- 趋势判断：上升趋势 / 下降趋势 / 震荡趋势
- 趋势强度：强 / 中 / 弱
```

**强制检查点**：必须完成趋势判断，缺失不得进入下一阶段。

---

### 4.2 动量指标分析

**任务**: 分析目标公司的动量指标。

**动量指标**：
- RSI（相对强弱指数）
- MACD（指数平滑异同移动平均线）
- 成交量变化

**执行步骤**：
1. 调用 `neodata-financial-search` skill
2. 搜索技术指标数据
3. 分析动量信号

**输出格式**：
```
【动量指标分析】
- RSI：[数值]（超买>70 / 超卖<30）
- MACD：[数值]（金叉/死叉）
- 成交量变化：[变化]%（放量/缩量）
- 动量信号：看多 / 看空 / 中性
```

**强制检查点**：必须完成动量分析，缺失不得进入下一阶段。

---

### 4.3 盈亏比计算

**任务**: 计算交易的盈亏比。

**盈亏比标准**：
- **盈亏比 ≥ 3:1** 才可执行交易（Stanley Druckenmiller标准）

**执行步骤**：
1. 确定目标价位（基于内在价值）
2. 确定止损价位（基于技术支撑）
3. 计算盈亏比：
   ```
   盈亏比 = (目标价 - 当前价) / (当前价 - 止损价)
   ```

**输出格式**：
```
【盈亏比计算】
- 当前股价：[股价]元
- 目标价位：[目标价]元（内在价值）
- 止损价位：[止损价]元（技术支撑）
- 预期收益：[收益]%（目标价 - 当前价）
- 最大损失：[损失]%（当前价 - 止损价）
- 盈亏比：[盈亏比]:1
- 是否满足标准（≥3:1）：[是/否]
```

**强制检查点**：
- 盈亏比 ≥ 3:1：进入下一阶段
- 盈亏比 < 3:1：**立即终止分析，输出"不投资"决策**

## Phase 5: 最终决策（强制执行）

### 5.1 综合评分汇总

**任务**: 汇总所有评分，生成最终决策。

**评分权重**：
- 宏观趋势评分：20%
- 行业趋势强度：30%
- 公司竞争优势：30%
- 技术趋势强度：20%

**执行步骤**：
1. 计算综合评分（加权平均）
2. 给出最终决策（BUY/SELL/HOLD）
3. 给出信心水平（高/中/低）

**输出格式**：
```
【综合评分汇总】
评分项          得分    权重    加权得分
宏观趋势评分    [得分]  20%     [加权得分]
行业趋势强度    [得分]  30%     [加权得分]
公司竞争优势    [得分]  30%     [加权得分]
技术趋势强度    [得分]  20%     [加权得分]
------------------------------------------------
综合评分        [综合评分]（满分10分）

【最终决策】
- 决策：BUY / SELL / HOLD
- 信心水平：高 / 中 / 低
- 建议买入价格：[价格]元/股（≤当前价）
- 建议仓位：[仓位]%（Kelly公式）
```

**强制检查点**：必须给出明确的 BUY/SELL/HOLD 决策，不得以"信号分歧"为由默认 HOLD。
