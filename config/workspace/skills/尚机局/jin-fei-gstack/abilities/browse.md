## 流程概览

browse 是持久化 headless Chromium，提供 ~100ms/命令的快速 QA 测试能力：

### 核心能力
1. **导航** — 打开 URL、前进/后退、刷新、等待加载
2. **交互** — 点击、填写、选择、上传、拖拽
3. **检查** — 验证元素、断言状态、检查文本
4. **视觉** — 截图、全页截图、对比差异
5. **快照** — 页面快照、DOM 分析、可访问性树
6. **标签页** — 打开/关闭/切换标签页
7. **服务器** — 启动本地服务器、检查端口

### 典型工作流
```
1. 启动浏览器（headed 或 headless）
2. 导航到目标 URL
3. 等待页面加载完成
4. 执行交互操作（点击、填写表单）
5. 验证结果（断言元素、检查文本）
6. 截图记录
7. 生成测试报告
```

## 关键特性

- **持久化会话**：浏览器实例跨命令保持，避免重复启动
- **快速响应**：~100ms/命令，适合迭代测试
- **Headed 模式**：支持可视化调试（macOS/Windows）
- **无头模式**：适合 CI/CD 环境（Linux）
- **快照系统**：页面快照 + DOM 分析 + 可访问性树

## 禁令

- ❌ 禁止在生产环境执行破坏性操作
- ❌ 禁止绕过登录验证（除非用户提供凭证）
- ❌ 禁止在无 headed 模式下处理需要人工验证的场景
