## 工作流概述

本工作流基于 Charlie Munger 的合规检查清单哲学，采用 **法律合规 + 财务合规 + 数据合规 + 知识产权合规 + 内部控制合规** 五大核心方法论。

**执行原则**：
1. **强制读完机制**：AI必须执行完整流程，不得跳过任何步骤
2. **合规率标准**：各项合规率≥95%
3. **风险等级**：风险等级≤低
4. **合规成本**：合规成本≤预算

## Phase 1: 合规检查目标明确（强制执行）

### 1.1 确定合规检查目标

**任务**: 明确合规检查的目标和范围。

**执行步骤**：
1. 明确合规检查目标：
   - 检查类型（法律/财务/数据/知识产权/内部控制）
   - 检查范围
   - 检查时间期限

**输出格式**：
```
【合规检查目标】
- 检查类型：[类型]
- 检查范围：[范围描述]
- 时间期限：[期限]
```

**强制检查点**：必须明确合规检查目标，缺失不得进入下一阶段。

## Phase 2: 法律合规检查（强制执行）

### 2.1 法律合规率评估

**任务**: 评估法律合规率。

**执行步骤**：
1. 检查法律合规项：
   - 公司法合规
   - 证券法合规
   - 劳动法合规
   - 税法合规
   - 行业法规合规

**输出格式**：
```

- 证券法合规：[是/否]
- 劳动法合规：[是/否]
- 税法合规：[是/否]
- 行业法规合规：[是/否]
- 法律合规率：[合规率]%
```

**强制检查点**：
- 法律合规率≥95%：进入下一阶段
- 法律合规率<95%：**立即整改后继续**

---

### 2.2 法律合规评分

**任务**: 对法律合规进行评分。

**评分标准**（0-10分）：
- **9-10分**：法律合规率≥99%，风险等级极低，合规成本低
- **7-8分**：法律合规率≥95%，风险等级低，合规成本中等
- **5-6分**：法律合规率≥90%，风险等级中等，合规成本较高
- **3-4分**：法律合规率<90%，风险等级高，合规成本高
- **0-2分**：法律合规失败，风险等级极高

**输出格式**：
```
【法律合规评分】
- 法律合规率评分：[评分]/10
- 法律风险评分：[评分]/10
- 法律合规总分：[总分]/10
```

**强制检查点**：
- 法律合规总分≥7分：进入下一阶段
- 法律合规总分<7分：**立即终止，输出"不通过"决策**

## Phase 6: 最终决策（强制执行）

**任务**: 汇总所有评分，生成最终决策。

**评分权重**：
- 法律合规：25%
- 财务合规：25%
- 数据合规：20%
- 知识产权合规：20%
- 内部控制合规：10%

**执行步骤**：
1. 计算综合评分（加权平均）
2. 给出最终决策（APPROVE/MODIFY/REJECT）
3. 给出信心水平（高/中/低）

**输出格式**：
```
【综合评分汇总】
评分项              得分    权重    加权得分
法律合规            [得分]  25%     [加权得分]
财务合规            [得分]  25%     [加权得分]
数据合规            [得分]  20%     [加权得分]
知识产权合规        [得分]  20%     [加权得分]
内部控制合规        [得分]  10%     [加权得分]
------------------------------------------------
综合评分            [综合评分]（满分10分）

【最终决策】
- 决策：APPROVE / MODIFY / REJECT
- 信心水平：高 / 中 / 低
- 合规率：[合规率]%
- 风险等级：[风险等级]
```

**强制检查点**：必须给出明确的 APPROVE/MODIFY/REJECT 决策。
