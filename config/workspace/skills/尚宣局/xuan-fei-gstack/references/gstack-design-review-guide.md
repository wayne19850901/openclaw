# 视觉 QA 完整流程

## 概述

gstack/design-review 是设计师之眼 QA 技能，发现视觉问题并修复。

## 问题检测清单

### 视觉不一致

| 检查项 | 说明 |
|--------|------|
| **颜色** | 同一元素在不同页面颜色是否一致 |
| **字体** | 字体家族、大小、粗细是否统一 |
| **间距** | 元素间距是否遵循设计系统 |
| **圆角** | 圆角半径是否一致 |
| **阴影** | 阴影样式是否统一 |

### 层级问题

| 检查项 | 说明 |
|--------|------|
| **标题大小** | H1 > H2 > H3 是否有足够对比 |
| **对比度** | 重要元素是否有足够视觉权重 |
| **留白** | 重要元素是否有足够留白 |

### AI slop 模式

| 模式 | 说明 |
|------|------|
| **过度通用** | "点击这里"、"了解更多" 等通用文案 |
| **缺乏品味** | 过度装饰、缺乏克制 |
| **模板化** | 一看就是 AI 生成的 |

### 慢交互

| 检查项 | 阈值 |
|--------|------|
| **响应时间** | >100ms |
| **动画时长** | >300ms |
| **加载时间** | >1s |

## 修复流程

```
1. 定位源代码（文件 + 行号）
2. 应用修复（Edit 工具）
3. 原子提交（git commit）
4. 验证修复（前后截图）
5. 重复直到问题清零
```
