# 翻译

## 概述

三模式翻译技能：**快速模式**直接翻译，**普通模式**分析后翻译，**精细模式**分析→翻译→审核→润色。

## 使用方式

```bash
/translate [--mode quick|normal|refined] [--to <lang>] <source>
```

## 三种模式

| 模式 | 步骤 | 适用场景 |
|------|------|----------|
| **快速** | 翻译 | 短文本、非正式内容 |
| **普通** | 分析→翻译 | 文章、博客、通用内容 |
| **精细** | 分析→翻译→审核→润色 | 出版级、重要文档 |

## 工作流程

### 步骤1：加载偏好

检查 EXTEND.md 配置：
- 默认目标语言
- 默认翻译模式
- 术语表

### 步骤2：创建输出目录

```
{source-dir}/{source-basename}-{target-lang}/
├── translation.md          # 最终翻译
├── 01-analysis.md          # 内容分析（普通/精细模式）
├── 02-prompt.md            # 翻译提示词
├── 03-draft.md             # 初稿（精细模式）
├── 04-critique.md          # 批评审核（精细模式）
├── 05-revision.md          # 修订版（精细模式）
└── chunks/                 # 长文本分块
```

### 步骤3：评估内容长度

| 内容长度 | 处理方式 |
|----------|----------|
| < 4000词 | 单次翻译 |
| >= 4000词 | 分块翻译（普通/精细模式） |

### 步骤4：翻译与精炼

**翻译原则**：
- **重写而非翻译**：用目标语言自然地重写，而非逐字翻译
- **准确性优先**：事实、数据、逻辑必须完全匹配原文
- **自然流畅**：使用目标语言的惯用语序
- **术语一致**：专业术语使用标准译法，首次出现标注原文

**精细模式额外步骤**：
- 批评审核：检查准确性、欧化语言、表达问题
- 修订润色：产出出版级译文

### 步骤5：输出

```
翻译完成（{mode}模式）

源文件：{source-path}
语言：{from} → {to}
输出目录：{output-dir}/
最终译文：{output-dir}/translation.md
应用术语：{count}条
```

## 翻译风格

| 风格 | 描述 |
|------|------|
| `storytelling` | 生动的叙事风格（默认） |
| `formal` | 专业、结构化 |
| `technical` | 精确、文档风格 |
| `literal` | 接近原文结构 |
| `academic` | 学术、严谨 |
| `business` | 简洁、结果导向 |
| `conversational` | 口语化、亲切 |

## 受众预设

| 受众 | 效果 |
|------|------|
| `general` | 普通读者，更多解释 |
| `technical` | 开发者，少注释技术术语 |
| `academic` | 学者，正式语体 |
| `business` | 商务人士，商务友好语调 |
