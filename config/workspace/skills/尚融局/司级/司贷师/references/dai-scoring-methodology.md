# DAI评分方法论（下行保护指数）

> **用途**: Mohnish Pabrai Dhandho投资核心方法论
> **适用场景**: 贷款审批、风险评估
> **更新时间**: 2026-05-21

---

## 一、DAI定义

**DAI（Downside Protection Index，下行保护指数）**：评估投资或贷款下行风险的量化指标，反映在最坏情况下的损失保护程度。

**核心原则**：
- 先考虑损失，再考虑收益
- 下行保护优先，上行收益次之
- DAI越高，风险越低

---

## 二、DAI计算公式

### 2.1 基础公式

```
DAI = (抵押覆盖率 × 0.4) + (现金流覆盖率 × 0.3) + (信用评分 × 0.2) + (行业稳定性 × 0.1)
```

### 2.2 权重说明

| 要素 | 权重 | 说明 |
|------|------|------|
| **抵押覆盖率** | 40% | 最重要，直接保护本金 |
| **现金流覆盖率** | 30% | 次重要，确保还款能力 |
| **信用评分** | 20% | 辅助，反映借款人诚信度 |
| **行业稳定性** | 10% | 补充，反映外部风险 |

---

## 三、DAI评分要素详解

### 3.1 抵押覆盖率（Collateral Coverage Ratio）

**定义**：抵押物估值与贷款金额的比率。

**计算公式**：
```
抵押覆盖率 = 抵押物估值 / 贷款金额
```

**评分标准**（0-10分）：
- **9-10分**：抵押覆盖率≥200%（强保护）
- **7-8分**：抵押覆盖率≥150%（中等保护）
- **5-6分**：抵押覆盖率≥120%（弱保护）
- **3-4分**：抵押覆盖率≥100%（最低保护）
- **0-2分**：抵押覆盖率<100%（无保护）

**案例**：
- 抵押物估值800万元，贷款金额500万元
- 抵押覆盖率 = 800万 / 500万 = 160%
- 评分：8/10（中等保护）

---

### 3.2 现金流覆盖率（Cash Flow Coverage Ratio）

**定义**：年净利润与年还款额的比率。

**计算公式**：
```
现金流覆盖率 = 年净利润 / 年还款额
```

**评分标准**（0-10分）：
- **9-10分**：现金流覆盖率≥200%（强还款能力）
- **7-8分**：现金流覆盖率≥150%（中等还款能力）
- **5-6分**：现金流覆盖率≥100%（弱还款能力）
- **3-4分**：现金流覆盖率≥80%（还款压力）
- **0-2分**：现金流覆盖率<80%（还款困难）

**案例**：
- 年净利润300万元，年还款额530万元（本金500万+利息30万）
- 现金流覆盖率 = 300万 / 530万 = 56.6%
- 评分：6/10（需调整贷款金额）

---

### 3.3 信用评分（Credit Score）

**定义**：借款人信用记录和诚信度的综合评分。

**评分标准**（0-10分）：
- **9-10分**：信用记录优秀，无逾期记录
- **7-8分**：信用记录良好，少量逾期
- **5-6分**：信用记录一般，有逾期记录
- **3-4分**：信用记录较差，多次逾期
- **0-2分**：信用记录极差，违约记录

**案例**：
- 借款人信用记录良好，无逾期记录
- 评分：8/10（信用良好）

---

### 3.4 行业稳定性（Industry Stability）

**定义**：借款人所在行业的稳定性和风险程度。

**评分标准**（0-10分）：
- **9-10分**：行业高度稳定（公用事业、必需消费品）
- **7-8分**：行业相对稳定（医疗、教育）
- **5-6分**：行业中度稳定（制造业、软件开发）
- **3-4分**：行业波动较大（周期性行业）
- **0-2分**：行业高度波动（新兴行业、高风险行业）

**案例**：
- 借款人所在行业：软件开发
- 评分：6/10（行业中度稳定）

## 四、DAI评分计算案例

### 4.1 案例：某科技公司贷款申请

**借款人信息**：
- 企业名称：某科技公司
- 行业：软件开发
- 申请金额：500万元
- 申请期限：12个月
- 年营收：2,000万元
- 年净利润：300万元
- 抵押物：商业房产，估值800万元
- 信用记录：良好

**DAI计算**：

| 要素 | 计算 | 得分 |
|------|------|------|
| **抵押覆盖率** | 800万 / 500万 = 160% | 8/10 |
| **现金流覆盖率** | 300万 / 530万 = 56.6% | 6/10 |
| **信用评分** | 信用记录良好 | 8/10 |
| **行业稳定性** | 软件开发行业 | 6/10 |

**DAI计算**：
```
DAI = (8 × 0.4) + (6 × 0.3) + (8 × 0.2) + (6 × 0.1)
    = 3.2 + 1.8 + 1.6 + 0.6
    = 7.2
```

**DAI评分：7.2** ✅（≥3.0，符合批准标准）

## 六、Kelly仓位管理

### 6.1 Kelly公式

**定义**：根据胜率和盈亏比计算最优仓位比例。

**计算公式**：
```
Kelly% = (p × b - q) / b
```

其中：
- p = 成功概率
- q = 失败概率（1 - p）
- b = 盈亏比

### 6.2 Kelly计算案例

**假设**：
- 胜率（p）：80%（基于DAI评分）
- 失败率（q）：20%
- 盈亏比（b）：2:1

**Kelly计算**：
```
Kelly% = (0.8 × 2 - 0.2) / 2
       = (1.6 - 0.2) / 2
       = 1.4 / 2
       = 0.7 = 70%
```

**保守调整**（使用半Kelly策略）：
```
最优贷款金额 = 总资金 × Kelly% × 0.5 = 500万 × 70% × 0.5 = 175万元
```

## 七、最佳实践

### 7.1 DAI评分流程

1. **收集借款人信息**：财务数据、抵押物信息、信用记录
2. **计算DAI评分**：使用公式计算DAI
3. **验证评分合理性**：检查数据准确性
4. **调整贷款金额**：根据Kelly公式调整
5. **给出最终决策**：APPROVE/REJECT/PENDING

### 7.2 常见误区

1. **过度依赖DAI**：DAI是参考指标，需结合定性分析
2. **忽视现金流覆盖率**：现金流是还款来源，需重点关注
3. **忽视抵押物估值波动**：抵押物估值可能下降，需保守评估
4. **忽视行业风险**：行业风险可能影响还款能力
