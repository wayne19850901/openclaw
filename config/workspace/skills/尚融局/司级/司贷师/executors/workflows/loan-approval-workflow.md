## 工作流概述

本工作流基于 Mohnish Pabrai 的 Dhandho 投资哲学，采用 **低风险高回报 + Kelly仓位管理 + 单笔损失限制** 三大核心方法论。

**执行原则**：
1. **强制读完机制**：AI必须执行完整流程，不得跳过任何步骤
2. **数据源唯一性**：所有金融数据必须使用 `neodata-financial-search` skill
3. **量化指标验证**：每个决策必须有量化指标支撑（DAI≥3.0、抵押覆盖率>150%、单笔损失≤2%）
4. **Kelly仓位管理**：必须使用Kelly公式计算最优仓位

## Phase 2: DAI评分（强制执行）

### 2.1 计算DAI（下行保护指数）

**任务**: 计算 DAI（Downside Protection Index）。

**DAI定义**：
- DAI = 抵押覆盖率 × 财务稳定性 × 业务可持续性
- **DAI ≥ 3.0** 才可批准贷款（Mohnish Pabrai标准）

**执行步骤**：
1. 计算抵押覆盖率：
   ```
   抵押覆盖率 = 抵押物估值 / 借款金额
   ```
2. 计算财务稳定性：
   ```
   财务稳定性 = 净资产 / 负债总额
   ```
3. 计算业务可持续性：
   ```
   业务可持续性 = 近3年平均净利润 / 近3年平均营业收入
   ```
4. 计算DAI：
   ```
   DAI = 抵押覆盖率 × 财务稳定性 × 业务可持续性
   ```

**输出格式**：
```
【DAI评分】
- 抵押覆盖率：[覆盖率]（抵押物估值/借款金额）
- 财务稳定性：[稳定性]（净资产/负债总额）
- 业务可持续性：[可持续性]（净利润/营业收入）
- DAI评分：[DAI]（三者乘积）
```

**强制检查点**：
- DAI ≥ 3.0：进入下一阶段
- DAI < 3.0：**立即终止审批，输出"不批准"决策**

## Phase 3: Kelly仓位管理（强制执行）

### 3.1 Kelly公式计算

**任务**: 使用Kelly公式计算最优贷款金额。

**Kelly公式**：
```
最优仓位比例 = (胜率 × 盈亏比 - 败率) / 盈亏比
```

**参数定义**：
- **胜率**：借款人按时还款的概率（基于历史数据）
- **盈亏比**：预期收益/预期损失
- **败率**：借款人违约的概率（1 - 胜率）

**执行步骤**：
1. 评估胜率（基于借款人历史还款记录）
2. 评估盈亏比（利息收益/本金损失）
3. 计算最优仓位比例
4. 计算最优贷款金额：
   ```
   最优贷款金额 = 总资金 × 最优仓位比例
   ```

**输出格式**：
```
【Kelly仓位计算】
- 胜率：[胜率]%（基于历史还款记录）
- 盈亏比：[盈亏比]（利息收益/本金损失）
- 最优仓位比例：[比例]%（Kelly公式）
- 总资金：[总资金]万元
- 最优贷款金额：[金额]万元
```

**强制检查点**：必须完成Kelly公式计算，缺失不得进入下一阶段。

## Phase 4: 现金流验证（强制执行）

### 4.1 现金流覆盖率计算

**任务**: 验证借款人的现金流是否足以覆盖还款。

**现金流覆盖率标准**：
- **现金流覆盖率 ≥ 1.5**（安全标准）

**执行步骤**：
1. 计算年还款金额：
   ```
   年还款金额 = 借款金额 / 借款期限 × 12
   ```
2. 计算现金流覆盖率：
   ```
   现金流覆盖率 = 年现金流 / 年还款金额
   ```

**输出格式**：
```
【现金流验证】
- 年现金流：[现金流]万元
- 年还款金额：[还款金额]万元
- 现金流覆盖率：[覆盖率]
- 是否满足标准（≥1.5）：[是/否]
```

**强制检查点**：
- 现金流覆盖率 ≥ 1.5：进入下一阶段
