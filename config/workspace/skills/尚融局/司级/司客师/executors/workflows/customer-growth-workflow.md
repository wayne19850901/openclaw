## 工作流概述

本工作流基于 Rakesh Jhunjhunwala 的"成长+安全边际"投资哲学，采用 **LTV/CAC分析 + 逆向思维获客 + 复利思维增长** 三大核心方法论。

**执行原则**：
1. **强制读完机制**：AI必须执行完整流程，不得跳过任何步骤
2. **数据源唯一性**：所有金融数据必须使用 `neodata-financial-search` skill
3. **量化指标验证**：每个决策必须有量化指标支撑（LTV/CAC≥3:1、NPS≥50%）
4. **逆向思维验证**：必须验证获客策略的逆向可行性（失败场景分析）

## Phase 1: 客户数据收集（强制执行）

### 1.1 获取目标公司客户数据

**任务**: 使用 `neodata-financial-search` skill 获取目标公司的客户相关数据。

**执行步骤**：
1. 调用 `neodata-financial-search` skill
2. 搜索目标公司的客户数据：
   - 客户数量
   - 客户增长率
   - 客户留存率
   - 客单价
   - 获客成本（CAC）
   - 客户生命周期价值（LTV）

**输出格式**：
```
【客户数据】
- 客户数量：[数量]
- 客户增长率：[增长率]%
- 客户留存率：[留存率]%
- 客单价：[客单价]元
- 获客成本（CAC）：[CAC]元
- 客户生命周期价值（LTV）：[LTV]元
```

**强制检查点**：必须获取以上6项客户数据，缺失任何一项不得进入下一阶段。

## Phase 2: LTV/CAC分析（强制执行）

### 2.1 计算LTV/CAC比率

**任务**: 计算客户生命周期价值与获客成本的比率，评估获客效率。

**计算公式**：
```
LTV/CAC比率 = LTV / CAC
```

**评估标准**：
- **优秀**：LTV/CAC ≥ 3:1（获客效率高）
- **良好**：LTV/CAC ≥ 2:1（获客效率中等）
- **不足**：LTV/CAC < 2:1（获客效率低）

**输出格式**：
```
【LTV/CAC分析】
- LTV/CAC比率：[比率]
- 评估结果：[优秀/良好/不足]
- 获客效率：[效率描述]
```

**强制检查点**：必须计算LTV/CAC比率并给出评估结果。

## Phase 3: 逆向思维获客验证（强制执行）

### 3.1 分析获客策略失败场景

**任务**: 使用逆向思维，分析获客策略可能失败的场景。

**分析维度**：
1. **获客渠道失败**：如果主要获客渠道失效，如何应对？
2. **客户流失风险**：如果客户留存率下降，如何应对？
3. **成本上升风险**：如果获客成本上升，如何应对？
4. **竞争加剧风险**：如果竞争对手获客策略更优，如何应对？

**输出格式**：
```
【逆向思维验证】
- 获客渠道失败应对：[应对方案]
- 客户流失风险应对：[应对方案]
- 成本上升风险应对：[应对方案]
- 竞争加剧风险应对：[应对方案]
```

**强制检查点**：必须分析4个失败场景并给出应对方案。

## Phase 4: 复利思维增长验证（强制执行）

### 4.1 计算复利增长潜力

**任务**: 使用复利思维，计算客户增长的长期潜力。

**计算公式**：
```
复利增长潜力 = 客户数量 × (1 + 客户增长率)^n
```

其中n为未来年数（通常5年）。

**评估标准**：
- **高潜力**：5年后客户数量增长≥100%
- **中潜力**：5年后客户数量增长≥50%
- **低潜力**：5年后客户数量增长<50%

**输出格式**：
```
【复利增长潜��】
- 当前客户数量：[数量]
- 客户增长率：[增长率]%
- 5年后客户数量：[数量]
- 增长倍数：[倍数]
- 增长潜力：[高/中/低]
```

**强制检查点**：必须计算5年复利增长潜力并给出评估结果。

## Phase 5: NPS与客户满意度验证（强制执行）

### 5.1 分析NPS（净推荐值）

**任务**: 分析客户满意度指标，验证客户忠诚度。

**评估标准**：
- **优秀**：NPS ≥ 50%（客户忠诚度高）
- **良好**：NPS ≥ 30%（客户忠诚度中等）
- **不足**：NPS < 30%（客户忠诚度低）

**输出格式**：
```
【NPS分析】
- NPS值：[数值]%
- 评估结果：[优秀/良好/不足]
- 客户忠诚度：[忠诚度描述]
```

**强制检查点**：必须分析NPS并给出评估结果。

## Phase 6: 最终决策与风险提示（强制执行）

### 6.1 综合评估与决策

**任务**: 综合所有分析结果，给出最终客户增长策略建议。

**决策标准**：
- **BUY（加强获客）**：LTV/CAC≥3:1、NPS≥50%、复利增长潜力高

- **SELL（优化获客）**：LTV/CAC<2:1、NPS<30%、复利增长潜力低

**输出格式**：
```

- LTV/CAC比率：[比率]
- NPS值：[数值]%

- 核心理由：[理由描述]

【操作建议】
- 获客策略：[策略建议]
- 留存策略：[策略建议]
- 成本控制：[策略建议]
- 竞争应对：[策略建议]
```

**强制检查点**：必须给出明确的BUY/HOLD/SELL决策和具体操作建议。
