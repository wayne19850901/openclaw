# 市场研究

## 概述

市场规模评估、竞争分析、机会验证。使用实用框架和免费数据源。

**来源**: 尚融局/market-research

## 使用方式

```bash
/market-research [研究主题]
```

## 核心框架

**研究前必做**：明确问题。"市场有多大？"无用。"在[地区]有多少[目标客户]愿意每年为[解决方案]支付[价格]？"可执行。

## 市场规模评估

| 方法 | 适用场景 | 方法 |
|------|----------|------|
| **自下而上** | 有单位经济数据 | 客户数 × 价格 × 频率 |
| **自上而下** | 探索新市场 | TAM → SAM → SOM |
| **可比公司** | 有类似产品 | 竞品收入 × 市场份额估算 |

**免费数据源**：政府人口普查、行业协会、上市公司财报、Statista 免费版、Google Trends、LinkedIn。

**红旗**：SAM 等于 TAM，说明没想清楚谁真正会买。

## 竞争分析

1. **绘制格局**：直接竞品、间接竞品、潜在进入者
2. **挖掘评价**：G2、Capterra、App Store — 反复出现的抱怨 = 机会
3. **追踪信号**：招聘信息（在做什么）、定价变化、功能发布
4. **评估护城河**：网络效应、转换成本、数据优势、监管壁垒

## 客户验证

**构建前**：与 20+ 潜在客户交谈。使用 Jobs-to-be-Done 或 Mom Test 框架。

**有意义的信号**：LOI、预付款、候补转注册转化、重复使用
**无意义的信号**："我会用的"、问卷热情、社交媒体点赞

## 常见陷阱

- **确认偏差**：只找支持论点的数据
- **幸存者偏差**：只研究成功公司
- **过时来源**：科技市场以月为单位变化
- **虚荣指标**：下载量、关注者、页面浏览量（无留存/收入）

## 边界

- **不提供投资建议**：不能基于市场研究推荐投资
- **不保证结果**：所有市场预测都是概率性的
- **转交专业人士**：统计有效的一手研究、受监管行业、深度竞争情报
