# Office Hours

## 概述

YC Office Hours 式创业咨询技能。六问框架验证创业想法，前提挑战测试信念，替代方案拓宽思路。

**来源**: gstack/office-hours

## 使用方式

```bash
/office-hours [--mode startup|builder] [问题描述]
```

## 双模式

| 模式 | 适用场景 | 核心问题 |
|------|----------|----------|
| **Startup** | 创业想法验证 | 六问框架 + 需求证据 |
| **Builder** | 技术/产品构建 | 问题陈述 + 约束条件 |

## 六问框架（Startup 模式）

1. **需求证据** — 具体用户行为证明需求存在
2. **现状** — 用户现在怎么解决
3. **目标用户** — 具体的人，不是类别
4. **最小付费版本** — 最小可售版本是什么
5. **约束条件** — 时间、资源、技术限制
6. **成功标准** — 可衡量的完成定义

## 工作流程

```
Phase 1: 模式选择
    ↓
Phase 2: 问题陈述（六问框架）
    ↓
Phase 3: 前提挑战
    ├── 识别隐含前提
    ├── 挑战每个前提
    └── 用户确认或反驳
    ↓
Phase 4: 替代方案
    ├── 方案 A: 渐进式
    ├── 方案 B: 颠覆式
    └── 外部设计声音（可选）
    ↓
Phase 5: 设计文档
    ├── 写入 ~/.gstack/projects/{slug}/
    ├── 对抗性评审
    └── 用户批准
    ↓
Phase 6: 创业资源推荐
    ├── Garry Tan 视频
    ├── YC Startup School
    ├── Paul Graham 文章
    └── Lightcone Podcast
```

## 执行指令

**加载原始 gstack SKILL.md**:

```bash
# 读取完整技能定义
source_content=$(cat ~/.claude/skills/gstack/office-hours/SKILL.md)
```

将 `source_content` 作为系统提示词的一部分，执行完整的 Office Hours 流程。

## 禁令

- **禁止开始实现** — 此技能产出设计文档，不写代码
- **禁止批量提问** — 一次只问一个问题
- **必须给出 Assignment** — 每次会话必须有具体行动建议
