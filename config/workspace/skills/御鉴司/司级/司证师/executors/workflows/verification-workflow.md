# 验证检查工作流

> **司证师** 执行工作流
> **方法论基础**: Charlie Munger 验证检查清单方法论
> **适用场景**: 结果验证、质量检查、合规审查
> **更新时间**: 2026-05-22

---

## Phase 1: 验证目标与准备

### 1.1 目标确认

明确本次验证检查的目标：
- 验证对象（结果/报告/流程/产品/决策）
- 验证范围（数据/结果/流程/质量/风险）
- 验证标准（行业/公司/项目标准）
- 验证结论要求（APPROVE/MODIFY/REJECT）

### 1.2 准备工作

| 准备项 | 要求 |
|--------|------|
| 验证材料 | 待验证文档/数据/结果齐全 |
| 验证标准 | 明确评分标准与阈值 |
| 验证工具 | 数据校验/比对/审计工具 |
| 验证人员 | 至少2人交叉验证 |

### 1.3 产出

- 验证计划书（含目标、范围、标准、时间表）

## Phase 2: 数据准确性验证

### 2.1 验证内容

验证数据是否准确、可靠：
- 数据准确性（数据值是否正确）
- 数据来源可靠性（数据来源是否可信）
- 数据一致性（不同来源数据是否一致）

### 2.2 验证方法

| 方法 | 说明 |
|------|------|
| 交叉比对 | 多来源数据交叉验证 |
| 逻辑校验 | 数据逻辑关系验证 |
| 抽样复算 | 随机抽样重新计算 |
| 时间校验 | 数据时效性验证 |

### 2.3 评分标准

| 指标 | 9-10分 | 7-8分 | 5-6分 | 3-4分 | 0-2分 |
|------|--------|-------|-------|-------|-------|
| 准确性 | ≥99% | ≥95% | ≥90% | <90% | 失败 |
| 来源可靠性 | 极高 | 高 | 中等 | 低 | 失败 |
| 一致性 | 极高 | 高 | 中等 | 低 | 失败 |

### 2.4 产出

- 数据准确性验证评分（0-10分）
- 数据验证报告（准确性 + 来源可靠性 + 一致性分析）

---

## Phase 3: 结果一致性验证

### 3.1 验证内容

验证结果是否一致、稳定：
- 结果一致性（多次计算结果是否一致）
- 结果稳定性（结果是否受输入微小变化影响）
- 结果可重复性（不同人员/时间/环境是否可复现）

### 3.2 验证方法

| 方法 | 说明 |
|------|------|
| 重复计算 | 独立重新计算验证 |
| 敏感性分析 | 输入微小变化对结果的影响 |
| 交叉验证 | 不同方法验证同一结果 |
| 历史比对 | 与历史结果对比验证 |

### 3.3 评分标准

| 指标 | 9-10分 | 7-8分 | 5-6分 | 3-4分 | 0-2分 |
|------|--------|-------|-------|-------|-------|
| 一致性 | ≥99% | ≥95% | ≥90% | <90% | 失败 |
| 稳定性 | 极高 | 高 | 中等 | 低 | 失败 |
| 可重复率 | ≥99% | ≥95% | ≥90% | <90% | 失败 |

### 3.4 产出

- 结果一致性验证评分（0-10分）
- 一致性验证报告（一致性 + 稳定性 + 可重复性分析）

## Phase 4: 流程合规性与质量标准验证

### 4.1 流程合规性验证

验证流程是否符合规范要求：
- 流程合规性（流程是否符合规定）
- 流程标准化（流程是否标准化）
- 流程可追溯性（流程是否可追溯）

### 4.2 质量标准验证

验证质量是否达到标准要求：
- 质量标准达成率（是否达到质量标准）
- 质量稳定性（质量是否稳定）
- 质量改进率（质量是否持续改进）

### 4.3 评分标准

**流程合规性**：

| 指标 | 9-10分 | 7-8分 | 5-6分 | 3-4分 | 0-2分 |
|------|--------|-------|-------|-------|-------|
| 合规性 | ≥99% | ≥95% | ≥90% | <90% | 失败 |
| 标准化 | 极高 | 高 | 中等 | 低 | 失败 |
| 可追溯性 | 极高 | 高 | 中等 | 低 | 失败 |

**质量标准**：

| 指标 | 9-10分 | 7-8分 | 5-6分 | 3-4分 | 0-2分 |
|------|--------|-------|-------|-------|-------|
| 达成率 | ≥99% | ≥95% | ≥90% | <90% | 失败 |
| 稳定性 | 极高 | 高 | 中等 | 低 | 失败 |
| 改进率 | ≥20% | ≥10% | ≥5% | <5% | 失败 |

### 4.4 产出

- 流程合规性验证评分（0-10分）
- 质量标准验证评分（0-10分）
- 流程验证报告 + 质量验证报告

## Phase 5: 风险控制验证

### 5.1 验证内容

验证风险控制是否有效：
- 风险识别率（风险是否全面识别，≥95%达标）
- 风险应对率（应对措施是否有效，≥95%达标）
- 风险控制效果（风险是否得到有效控制）

### 5.2 验证方法

| 方法 | 说明 |
|------|------|
| 风险清单审查 | 审查风险识别完整性 |
| 应对措施验证 | 验证应对措施有效性 |
| 残余风险评估 | 评估残余风险水平 |
| 风险趋势分析 | 分析风险变化趋势 |

### 5.3 评分标准

| 指标 | 9-10分 | 7-8分 | 5-6分 | 3-4分 | 0-2分 |
|------|--------|-------|-------|-------|-------|
| 识别率 | ≥99% | ≥95% | ≥90% | <90% | 失败 |
| 应对率 | ≥99% | ≥95% | ≥90% | <90% | 失败 |
| 控制效果 | 极好 | 好 | 一般 | 差 | 失败 |

### 5.4 产出

- 风险控制验证评分（0-10分）
- 风险验证报告（识别率 + 应对率 + 控制效果分析）

## Phase 6: 综合评分与通过建议

### 6.1 综合评分计算

```
验证检查总分 = (数据准确性验证 + 结果一致性验证 + 流程合规性验证 + 质量标准验证 + 风险控制验证) / 5
```

### 6.2 通过决策标准

| 总分 | 等级 | 通过建议 | 说明 |
|------|------|----------|------|
| **≥8分** | 优秀 | **APPROVE（批准）** | 各项达标，可以优先通过 |
| **6-8分** | 良好 | **MODIFY（修改）** | 部分不达标，需修改后通过 |
| **<6分** | 较差 | **REJECT（拒绝）** | 大部分不达标，不建议通过 |

### 6.3 风险警示

- 数据准确性不达标 → 可能导致数据错误
- 结果一致性不达标 → 可能导致结果不稳定
- 流程合规性不达标 → 可能导致流程违规
- 质量标准不达标 → 可能导致质量问题
- 风险控制不达标 → 可能导致风险失控

### 6.4 最终产出

| 产出项 | 内容 |
|--------|------|
| 验证检查总分 | X.X/10 |
| 各维度评分 | 数据准确性/结果一致性/流程合规/质量标准/风险控制 |
| 通过建议 | APPROVE / MODIFY / REJECT |
| 关键问题 | Top 3 不达标项 |
| 整改建议 | 具体可执行的整改措施 |
| 验证结论 | 明确的批准/修改/拒绝决定 |
