## 工作流概述

本工作流基于 Nassim Taleb 的反脆弱哲学，采用 **压力测试设计 + 极端场景分析 + 凸性防御策略** 三大核心方法论。

**执行原则**：
1. **强制读完机制**：AI必须执行完整流程，不得跳过任何步骤
2. **数据源唯一性**：所有金融数据必须使用 `neodata-financial-search` skill
3. **量化指标验证**：每个决策必须有量化指标支撑（反脆弱指数≥7分、压力测试通过率≥80%）
4. **极端场景验证**：必须验证极端场景下的生存能力

## Phase 2: 压力测试设计（强制执行）

### 2.1 设计极端场景压力测试

**任务**: 设计极端场景，测试目标公司的生存能力。

**压力测试场景**（5个极端场景）：
1. **收入下降50%**：测试公司在收入大幅下降情况下的生存能力
2. **成本上升30%**：测试公司在成本大幅上升情况下的盈利能力
3. **现金流断裂3个月**：测试公司在现金流断裂情况下的生存能力
4. **市场份额下降20%**：测试公司在市场份额大幅下降情况下的竞争力
5. **关键人员流失**：测试公司在关键人员流失情况下的运营能力

**通过标准**：
- **优秀**：通过5个场景（100%）
- **良好**：通过4个场景（≥80%）
- **一般**：通过3个场景（≥60%）
- **不足**：通过<3个场景（<60%）

**输出格式**：
```
【压力测试设计】
- 场景1（收入下降50%）：[通过/失败] - [应对方案]
- 场景2（成本上升30%）：[通过/失败] - [应对方案]
- 场景3（现金流断裂3个月）：[通过/失败] - [应对方案]
- 场景4（市场份额下降20%）：[通过/失败] - [应对方案]
- 场景5（关键人员流失）：[通过/失败] - [应对方案]
- 通过率：[通过率]%
- 评估结果：[优秀/良好/一般/不足]
```

**强制检查点**：必须完成5个极端场景压力测试并给出通过率。

## Phase 4: 凸性防御策略验证（强制执行）

### 4.1 评估凸性防御能力

**任务**: 评估目标公司的凸性防御能力（从极端事件中获益的能力）。

**评估维度**：
1. **上行空间**：极端正面事件带来的收益潜力
2. **下行保护**：极端负面事件带来的损失限制
3. **不对称收益**：上行空间与下行保护的不对称性
4. **适应能力**：从极端事件中恢复并更强的能力

**评分标准**（0-10分）：
- **强凸性**：凸性指数≥8分（从极端事件中显著获益）
- **弱凸性**：凸性指数≥7分（从极端事件中适度获益）
- **脆弱性**：凸性指数≥6分（在极端事件中受损有限）
- **强脆弱**：凸性指数<6分（在极端事件中严重受损）

**计算公式**：
```
凸性指数 = (上行空间评分 + 下行保护评分 + 不对称收益评分 + 适应能力评分) / 4
```

**输出格式**：
```
【凸性防御策略验证】
- 上行空间评分：[评分]/10
- 下行保护评分：[评分]/10
- 不对称收益评分：[评分]/10
- 适应能力评分：[评分]/10
- 凸性指数：[指数]/10
- 评估结果：[强凸性/弱凸性/脆弱性/强脆弱]
```

**强制检查点**：必须评估凸性防御能力并给出评分。

## Phase 5: 反脆弱指数综合评分（强制执行）

### 5.1 计算反脆弱指数总分

**任务**: 综合所有分析维度，计算反脆弱指数总分。

**计算公式**：
```
反脆弱指数 = (压力测试通过率 × 0.4) + (凸性指数 × 0.4) + (极端场景影响评分 × 0.2)
```

**评分标准**（0-10分）：
- **强反脆弱**：反脆弱指数≥8分
- **弱反脆弱**：反脆弱指数≥7分
- **脆弱**：反脆弱指数≥6分
- **强脆弱**：反脆弱指数<6分

**输出格式**：
```
【反脆弱指数综合评分】
- 压力测试通过率：[通过率]%
- 凸性指数：[指数]/10
- 极端场景影响评分：[评分]/10
- 反脆弱指数：[指数]/10
- 评估结果：[强反脆弱/弱反脆弱/脆弱/强脆弱]
```

**强制检查点**：必须计算反脆弱指数并给出评估结果。

## Phase 6: 最终决策与风险提示（强制执行）

### 6.1 综合评估与决策

**任务**: 综合所有分析结果，给出最终安全投资建议。

**决策标准**：
- **BUY（安全投资）**：反脆弱指数≥8分、压力测试通过率≥80%、凸性指数≥7分

**输出格式**：
```

- 凸性指数：[指数]/10
- 核心理由：[理由描述]

【操作建议】
- 安全策略：[策略建议]
```

**强制检查点**：必须给出明确的BUY/HOLD/SELL决策和具体操作建议。
