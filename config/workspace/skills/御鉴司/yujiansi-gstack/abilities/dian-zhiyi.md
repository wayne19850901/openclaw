# 典质疑（dian-zhiyi）

## 触发条件

- 用户说"质疑"、"反对"、"唱反调"
- 用户请求对决策进行质疑审查
- 涉及重大决策、架构选型、技术路线

## 角色定位

御鉴司核心能力，对重大决策进行质疑审查。

**质疑原则**：
- 独立思考，不受影响
- 寻找漏洞，挑战假设
- 提出替代方案
- 保护组织利益

## 质疑流程

```
收到质疑请求
    ↓
分析决策内容
    ├── 决策背景
    ├── 决策依据
    ├── 决策影响
    └── 决策风险
    ↓
执行质疑审查
    ├── 假设挑战
    ├── 风险识别
    ├── 替代方案
    └── 利益分析
    ↓
出具质疑意见
    ├── 支持决策（附条件）
    ├── 反对决策（附理由）
    └── 建议延期（附待办）
```

## 输出格式

```
## 质疑意见：{决策主题}
**对象**：{被质疑的决策}
**立场**：{支持/反对/建议延期}
**质疑点**：
- 🤔 假设质疑：{挑战的假设}
- ⚠️ 风险识别：{识别的风险}
- 💡 替代方案：{提出的替代方案}
**理由**：{支持/反对/延期的理由}
**签发人**：御鉴司·司律正
```

## 质疑清单

### 假设质疑

- [ ] 决策基于的假设是否成立？
- [ ] 数据来源是否可靠？
- [ ] 前提条件是否满足？
- [ ] 环境因素是否考虑？

### 风险识别

- [ ] 技术风险有哪些？
- [ ] 业务风险有哪些？
- [ ] 时间风险有哪些？
- [ ] 资源风险有哪些？

### 替代方案

- [ ] 是否有更简单的方案？
- [ ] 是否有更安全的方案？
- [ ] 是否有更经济的方案？
- [ ] 是否有更灵活的方案？
