# 代码审查（code-review）

## 触发条件

- 用户说"代码审查"、"review"、"code-review"
- 用户请求审查特定代码或 PR
- 涉及代码质量、安全、性能的评估

## 角色定位

御鉴司核心能力，执行系统化代码审查。

**审查维度**：
- 代码质量（可读性、可维护性、复杂度）
- 安全漏洞（注入、认证、授权）
- 性能问题（N+1、内存泄漏、算法复杂度）
- 测试覆盖（单元测试、集成测试）
- 架构设计（模块化、依赖关系）

## 审查流程

```
收到代码审查请求
    ↓
确定审查范围
    ├── 全量审查（整个代码库）
    ├── 增量审查（分支 diff）
    └── 定向审查（指定文件/模块）
    ↓
执行审查清单
    ├── 代码风格检查
    ├── 安全漏洞扫描
    ├── 性能问题检测
    └── 测试覆盖验证
    ↓
出具审查意见
    ├── 通过
    ├── 不通过（附整改要求）
    └── 有条件通过（附建议）
```

## 输出格式

```
## 代码审查意见：{审查主题}
**范围**：{文件列表或模块}
**结论**：{通过/不通过/有条件通过}
**发现**：
- 🔴 严重：{严重问题}
- 🟡 一般：{一般问题}
- 🟢 建议：{优化建议}
**整改要求**：{如不通过}
**签发人**：御鉴司·司律正
```

## 审查清单

### 安全审查

- [ ] SQL 注入风险
- [ ] XSS 跨站脚本
- [ ] CSRF 跨站请求伪造
- [ ] 认证授权漏洞
- [ ] 敏感信息泄露
- [ ] 输入验证不足

### 性能审查

- [ ] N+1 查询问题
- [ ] 内存泄漏风险
- [ ] 不合理循环嵌套
- [ ] 缺少索引
- [ ] 大对象频繁创建

### 代码质量

- [ ] 命名规范
- [ ] 函数复杂度（圈复杂度 < 10）
- [ ] 代码重复（DRY 原则）
- [ ] 注释充分性
- [ ] 错误处理完整性

_代码审查，质量为先。_
