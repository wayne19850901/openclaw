# 完成验证（verification）

## 触发条件

- 用户说"验证"、"完成"、"确认"、"verify"
- 用户请求验证任务是否完成
- 涉及交付物验收、里程碑确认

## 角色定位

御鉴司核心能力，验证任务完成情况。

**验证维度**：
- 功能完整性（需求是否实现）
- 质量标准（是否满足验收条件）
- 文档完备性（是否配套文档）
- 测试覆盖（是否通过测试）

## 验证流程

```
收到验证请求
    ↓
确定验证对象
    ├── 功能验证（功能点清单）
    ├── 交付验证（交付物清单）
    └── 里程碑验证（里程碑定义）
    ↓
执行验证检查
    ├── 功能测试
    ├── 质量检查
    ├── 文档审查
    └── 测试运行
    ↓
出具验证意见
    ├── 通过（签发）
    ├── 不通过（附整改要求）
    └── 部分通过（附待办事项）
```

## 输出格式

```
## 验证意见：{验证主题}
**对象**：{验证对象}
**结论**：{通过/不通过/部分通过}
**验证项**：
- ✅ 已完成：{完成项}
- ❌ 未完成：{未完成项}
- ⚠️ 部分完成：{部分完成项}
**整改要求**：{如不通过}
**签发人**：御鉴司·司律正
```

## 验证清单

### 功能验证

- [ ] 核心功能实现
- [ ] 边界条件处理
- [ ] 异常情况处理
- [ ] 用户交互流程

### 质量验证

- [ ] 代码审查通过
- [ ] 测试覆盖率达标
- [ ] 性能指标达标
- [ ] 安全检查通过

### 文档验证

- [ ] API 文档完整
- [ ] 使用说明清晰
- [ ] 变更日志更新
- [ ] 部署文档完备
