# 课程框架参考

## ADDIE 教学设计模型

ADDIE 是经典的系统化教学设计模型，包含五个阶段：

| 阶段 | 英文 | 内容 | 关键活动 |
|------|------|------|----------|
| 分析 | Analysis | 学习者需求分析 | 需求调研、知识缺口识别、学习者画像 |
| 设计 | Design | 学习目标与结构设计 | 目标设定（Bloom）、内容结构、评估标准 |
| 开发 | Development | 教学材料制作 | 教案编写、课件制作、练习题设计 |
| 实施 | Implementation | 教学交付 | 授课、辅导、答疑 |
| 评估 | Evaluation | 效果评估 | 测试、反馈收集、改进建议 |

---

## Bloom 学习目标分类法

Bloom 分类法将学习目标分为六个层级，从低到高：

| 层级 | 认知过程 | 动词示例 | 教学设计要点 |
|------|----------|----------|-------------|
| **记忆** | Recall | 定义、列举、识别 | 提供记忆线索、重复强化 |
| **理解** | Comprehend | 解释、描述、总结 | 提供类比、可视化、举例 |
| **应用** | Apply | 使用、计算、演示 | 提供练习场景、步骤指导 |
| **分析** | Analyze | 比较、区分、组织 | 提供对比材料、结构化任务 |
| **评价** | Evaluate | 评判、论证、选择 | 提供评价标准、多方案对比 |
| **创造** | Create | 设计、构建、规划 | 提供创作空间、开放性任务 |

**设计原则**：课程目标应覆盖多个层级，避免只停留在记忆/理解层。

## 课程结构模板

### 基础结构

```markdown
# [课程名称]

## 课程信息
- **目标学员**：[学员画像描述]
- **前置知识**：[必要基础]
- **课程时长**：[X 课时 / X 周]
- **学习目标**：
  - [Bloom 层级]：[具体目标1]
  - [Bloom 层级]：[具体目标2]

## 模块大纲

### 模块一：[模块名]
- **知识点**：[列表]
- **教学活动**：讲授 / 练习 / 讨论 / 项目
- **评估方式**：[测验 / 作品 / 反思]
- **资源**：教程 / 参考卡 / 操作指南

### 模块二：[模块名]
...
```

### 进阶结构（含 Diataxis 覆盖）

```markdown
# [课程名称]

## 课程信息
...

## Diataxis 文档覆盖

|------|----------|--------|-----------|-------------|
| 模块一 | ✅ 入门教程 | ✅ 安装指南 | ✅ 语法速查 | ✅ 概念解析 |
| 模块二 | ✅ 进阶教程 | ❌ 待补充 | ✅ API 参考 | ✅ 原理说明 |

```

## 教学活动类型

| 类型 | 适用场景 | 设计要点 |
|------|----------|----------|
| **讲授** | 概念引入、原理讲解 | 时间控制、可视化、互动提问 |
| **演示** | 操作流程、工具使用 | 步骤清晰、预期结果明确 |
| **练习** | 知识点巩固 | 难度梯度、即时反馈 |
| **讨论** | 深度理解、观点碰撞 | 引导性问题、小组分工 |
| **项目** | 综合应用、创造产出 | 真实场景、开放性、成果展示 |

---

## 评估方式

| 方式 | 适用层级 | 特点 |
|------|----------|------|
| **测验** | 记忆/理解/应用 | 快速、标准化、客观评分 |
| **作品** | 应用/分析/创造 | 综合能力、主观评分、需评分标准 |
| **反思** | 理解/评价 | 元认知、自我评估、需引导问题 |
| **同伴互评** | 分析/评价 | 多视角、批判性思维、需评分标准 |

## 与 gstack 联动

### ADDIE × gstack

| ADDIE 阶段 | gstack 能力 | 用途 |
|------------|-------------|------|
| Analysis | learn | 检索历史教学经验、学习者画像 |
| Design | document-generate | 生成课程大纲、学习目标文档 |
| Development | document-generate | 生成教程/指南/参考卡 |
| Implementation | post-to-wechat | 发布教学内容 |
| Evaluation | learn | 沉淀评估经验、反馈记录 |

### 课程设计工作流

```
learn（检索历史经验）
  ↓
course-design（ADDIE 分析+设计）
  ↓
document-generate（ADDIE 开发：生成各类文档）
  ↓
document-release（发布前审核覆盖度）
  ↓
post-to-wechat（ADDIE 实施：发布内容）
  ↓
learn（ADDIE 评估：沉淀经验）
```
