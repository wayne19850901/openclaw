# 教程写作

## 概述

撰写教育性教程内容。与 `document-generate` 的区别：本能力专注于教学视角的
内容创作，而 document-generate 侧重于技术文档的结构化生成。

## 触发词

- 教程写作
- 编写教程
- 写教学文章
- 教学内容创作

## 教程写作原则

### 1. 学习者中心

- 从学习者已知出发，不假设背景知识
- 每步操作都有预期结果
- 错误处理与常见陷阱提前说明

### 2. 渐进式结构

```
基础概念 → 简单示例 → 练习巩固 → 进阶应用 → 综合项目
```

### 3. 多模态表达

| 模态 | 适用场景 |
|------|----------|
| 文字说明 | 概念解释、原理阐述 |
| 代码示例 | 编程教学、工具使用 |
| 图表 | 流程、架构、对比 |
| 练习题 | 知识点巩固 |

## 教程模板

```markdown
# [教程标题]

## 你将学到什么
- [学习目标1]
- [学习目标2]

## 前置知识
- [要求1]
- [要求2]

## 正文

### 第一步：[标题]
**做什么**：[操作描述]
**为什么**：[原理解释]
**怎么做**：
```代码或步骤```
**预期结果**：[应该看到什么]

### 常见问题
- **问题1**：[描述] → [解决方案]

## 小结
- [要点回顾]

## 下一步
- [进阶方向]
- [相关教程链接]
```

## 与 document-generate 的分工

| 场景 | 使用能力 | 原因 |
|------|----------|------|
| 技术项目缺文档 | document-generate | 需要 Diataxis 结构化覆盖 |
| 教学课程写教程 | tutorial-writing | 需要教学法和学习者视角 |
| 两者重叠 | tutorial-writing + document-generate | 教学视角 + 结构化覆盖 |

## 与 gstack 联动

```
教程写作流程：
  learn（检索相关教学经验）
    → tutorial-writing（撰写教程内容）
    → document-release（发布前审核覆盖度）
    → post-to-wechat（发布到公众号）
    → learn（沉淀写作经验）
```
