# 课程设计

## 概述

设计教学大纲、课程结构、学习目标。结合 gstack 知识管理能力，让课程设计基于
经验沉淀而非凭空想象。

## 触发词

- 课程设计
- 教学设计
- 教学大纲
- 课程结构
- 学习目标

## 课程设计框架

### ADDIE 模型

| 阶段 | 内容 | gstack 联动 |
|------|------|-------------|
| Analysis（分析） | 学习者需求分析、知识缺口识别 | learn（检索历史经验） |
| Design（设计） | 学习目标、评估标准、内容结构 | document-generate（生成课程大纲） |
| Development（开发） | 教学材料制作 | document-generate（生成教程/指南） |
| Implementation（实施） | 教学交付 | post-to-wechat（发布内容） |
| Evaluation（评估） | 效果评估、反馈收集 | learn（沉淀评估经验） |

### 学习目标设计（Bloom 分类法）

| 层级 | 动词 | 示例 |
|------|------|------|
| 记忆 | 定义、列举、识别 | 定义什么是递归 |
| 理解 | 解释、描述、总结 | 解释递归的执行过程 |
| 应用 | 使用、计算、演示 | 使用递归解决阶乘问题 |
| 分析 | 比较、区分、组织 | 比较递归与迭代的优劣 |
| 评价 | 评判、论证、选择 | 选择最适合场景的算法 |
| 创造 | 设计、构建、规划 | 设计一个递归数据结构 |

## 课程结构模板

```markdown
# [课程名称]

## 课程信息
- 目标学员：[学员画像]
- 前置知识：[要求]
- 课程时长：[X 课时]
- 学习目标：[Bloom 层级目标]

## 模块一：[模块名]
### 知识点
- [知识点1]
- [知识点2]

### 教学活动
- 讲授：[内容]
- 练习：[任务]
- 评估：[方式]

### 资源
- 教程：[document-generate: tutorial]
- 参考卡：[document-generate: reference]
- 操作指南：[document-generate: how-to]
```

## 与 gstack 联动

```
课程设计流程：
  learn（检索历史教学经验）
    → course-design（设计课程结构）
    → document-generate（生成各类型教学文档）
    → document-release（发布前审核）
    → post-to-wechat（发布到公众号）
    → learn（沉淀本次设计经验）
```
