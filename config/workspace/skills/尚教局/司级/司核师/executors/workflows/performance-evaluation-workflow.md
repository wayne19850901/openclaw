## 工作流概述

本工作流基于 Charlie Munger 的绩效评估清单哲学，采用 **目标达成 + 工作质量 + 工作效率 + 团队协作 + 创新能力** 五大核心方法论。

**执行原则**：
1. **强制读完机制**：AI必须执行完整流程，不得跳过任何步骤
2. **目标达成标准**：目标达成率≥80%
3. **工作质量标准**：工作质量评分≥7分
4. **工作效率标准**：工作效率评分≥7分

---

## Phase 1: 绩效评估目标明确（强制执行）

### 1.1 确定绩效评估目标

**任务**: 明确绩效评估的目标和范围。

**执行步骤**：
1. 明确绩效评估目标：
   - 评估类型（个人/团队/项目）
   - 评估范围
   - 评估时间期限

**输出格式**：
```
【绩效评估目标】
- 评估类型：[类型]
- 评估范围：[范围描述]
- 时间期限：[期限]
```

**强制检查点**：必须明确绩效评估目标，缺失不得进入下一阶段。

---

## Phase 2: 目标达成评估（强制执行）

### 2.1 目标达成率评估

**任务**: 评估目标达成情况。

**执行步骤**：
1. 统计目标达成情况：
   - 目标总数
   - 达成目标数
   - 目标达成率

**输出格式**：
```
【目标达成评估】
- 目标总数：[数量]个
- 达成目标数：[数量]个
- 目标达成率：[达成率]%
```

**强制检查点**：
- 目标达成率≥80%：进入下一阶段
- 目标达成率<80%：**记录原因后继续**

---

### 2.2 目标达成评分

**任务**: 对目标达成进行评分。

**评分标准**（0-10分）：
- **9-10分**：目标达成率≥95%，质量极高，时间提前
- **7-8分**：目标达成率≥80%，质量高，时间按计划
- **5-6分**：目标达成率≥60%，质量中等，时间延后
- **3-4分**：目标达成率<60%，质量低，时间严重延后
- **0-2分**：目标达成失败，无法达成

**输出格式**：
```
【目标达成评分】
- 目标达成率评分：[评分]/10
- 目标达成质量评分：[评分]/10
- 目标达成总分：[总分]/10
```

**强制检查点**：
- 目标达成总分≥6分：进入下一阶段
- 目标达成总分<6分：**立即终止，输出"不合格"决策**

## Phase 6: 最终决策（强制执行）

### 6.1 绩效评估评分汇总

**任务**: 汇总所有评分，生成最终决策。

**评分权重**：
- 目标达成：30%
- 工作质量：25%
- 工作效率：20%
- 团队协作：15%
- 创新能力：10%

**执行步骤**：
1. 计算综合评分（加权平均）
2. 给出最终决策（EXCELLENT/GOOD/AVERAGE/POOR）
3. 给出信心水平（高/中/低）

**输出格式**：
```
【综合评分汇总】
评分项          得分    权重    加权得分
目标达成        [得分]  30%     [加权得分]
工作质量        [得分]  25%     [加权得分]
工作效率        [得分]  20%     [加权得分]
团队协作        [得分]  15%     [加权得分]
创新能力        [得分]  10%     [加权得分]
------------------------------------------------
综合评分        [综合评分]（满分10分）

【最终决策】
- 决策：EXCELLENT / GOOD / AVERAGE / POOR
- 信心水平：高 / 中 / 低
- 绩效等级：[等级]
```

**强制检查点**：必须给出明确的 EXCELLENT/GOOD/AVERAGE/POOR 决策。
