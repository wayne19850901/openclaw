## 工作流概述

本工作流基于工具集成专家的 MCP Server 设计哲学，采用 **工具接口设计 + 数据格式统一 + 错误处理规范 + 性能优化 + 安全性设计** 五大核心方法论。

**执行原则**：
1. **强制读完机制**：AI必须执行完整流程，不得跳过任何步骤
2. **工具接口标准化**：所有工具接口必须符合 MCP 协议
3. **数据格式统一性**：数据格式必须统一为 JSON
4. **错误处理规范性**：错误处理必须包含错误码和错误消息

---

## Phase 1: 需求收集（强制执行）

### 1.1 获取工具集成需求

**任务**: 收集工具集成的详细需求。

**执行步骤**：
1. 分析用户需求：
   - 工具名称
   - 工具功能
   - 输入参数
   - 输出格式
   - 错误处理需求

**输出格式**：
```
【工具集成需求】
- 工具名称：[名称]
- 工具功能：[功能描述]
- 输入参数：[参数列表]
- 输出格式：[格式描述]
- 错误处理需求：[需求描述]
```

**强制检查点**：必须获取以上5项需求，缺失任何一项不得进入下一阶段。

---

### 1.2 获取技术约束

**任务**: 获取工具集成的技术约束。

**执行步骤**：
1. 分析技术约束：
   - 性能要求（响应时间）
   - 安全性要求（数据加密）
   - 可扩展性要求（并发处理）
   - 兼容性要求（平台兼容）

**输出格式**：
```
【技术约束】
- 性能要求：响应时间≤[时间]ms
- 安全性要求：[安全级别]
- 可扩展性要求：并发数≥[数量]
- 兼容性要求：[平台列表]
```

**强制检查点**：必须获取技术约束，缺失不得进入下一阶段。

---

## Phase 2: 工具接口设计（强制执行）

### 2.1 接口参数设计

**任务**: 设计工具接口的输入参数和输出格式。

**执行步骤**：
1. 设计输入参数：
   - 必填参数
   - 可选参数
   - 参数类型
   - 参数验证规则

2. 设计输出格式：
   - 输出字段
   - 输出类型
   - 输出结构

**输出格式**：
```
【接口参数设计】
输入参数：
- [参数名]（必填）：[类型]，[验证规则]
- [参数名]（可选）：[类型]，[验证规则]

输出格式：
{
  "success": true/false,
  "data": { [输出数据] },
  "error": { "code": "[错误码]", "message": "[错误消息]" }
}
```

**强制检查点**：必须完成接口参数设计，缺失不得进入下一阶段。

---

### 2.2 接口评分（MCP Server评分）

**任务**: 对接口设计进行评分（基于MCP Server评分标准）。

**评分标准**（0-10分）：
- **9-10分**：接口设计完整，参数清晰，输出格式统一
- **7-8分**：接口设计良好，参数基本完整，输出格式规范
- **5-6分**：接口设计一般，参数部分缺失，输出格式不统一
- **3-4分**：接口设计较差，参数不完整，输出格式混乱
- **0-2分**：接口设计失败，无法正常调用

**输出格式**：
```
【接口评分】
- 工具接口设计评分：[评分]/10
- 数据格式统一评分：[评分]/10
- 错误处理机制评分：[评分]/10
- 性能优化评分：[评分]/10
- 安全性设计评分：[评分]/10
- MCP Server总分：[总分]/10
```

**强制检查点**：
- MCP Server总分 ≥ 6分：进入下一阶段
- MCP Server总分 < 6分：**立即终止，输出"不集成"决策**

---

## Phase 3: 数据格式统一（强制执行）

### 3.1 数据格式设计

**任务**: 设计统一的数据格式（JSON）。

**执行步骤**：
1. 设计数据结构
2. 定义字段命名规范
3. 定义数据类型
4. 定义数据验证规则

**输出格式**：
```
【数据格式设计】
{
  "version": "1.0",
  "timestamp": "[时间戳]",
  "data": {
    [数据字段]
  },
  "metadata": {
    [元数据]
  }
}
```

**强制检查点**：必须完成数据格式设计，缺失不得进入下一阶段。

---

## Phase 4: 错误处理机制（强制执行）

### 4.1 错误码设计

**任务**: 设计统一的错误码和错误消息。

**执行步骤**：
1. 设计错误码范围：
   - 1000-1999：客户端错误
   - 2000-2999：服务端错误
   - 3000-3999：网络错误
   - 4000-4999：数据错误

2. 设计错误消息格式：
   - 错误码
   - 错误消息
   - 解决方案

**输出格式**：
```
【错误码设计】
错误码范围：
- 1000-1999：客户端错误
  - 1001：参数缺失
  - 1002：参数格式错误
- 2000-2999：服务端错误
  - 2001：服务不可用
  - 2002：处理超时
```

**强制检查点**：必须完成错误码设计，缺失不得进入下一阶段。

---

## Phase 5: 性能优化（强制执行）

### 5.1 性能测试

**任务**: 测试工具接口的性能。

**执行步骤**：
1. 测试响应时间（目标：<1s）
2. 测试并发处理能力（目标：≥100并发）
3. 测试资源利用率（目标：CPU<70%，内存<80%）

**输出格式**：
```
【性能测试结果】
- 平均响应时间：[时间]ms
- 峰值响应时间：[时间]ms
- 并发处理能力：[并发数]
- CPU利用率：[利用率]%
- 内存利用率：[利用率]%
```

**强制检查点**：
- 响应时间 < 1s：进入下一阶段
- 响应时间 ≥ 1s：**优化后重新测试**

---

## Phase 6: 最终决策（强制执行）

### 6.1 综合评分汇总

**任务**: 汇总所有评分，生成最终决策。

**评分权重**：
- 工具接口设计：30%
- 数据格式统一：20%
- 错误处理机制：20%
- 性能优化：20%
- 安全性设计：10%

**执行步骤**：
1. 计算综合评分（加权平均）
2. 给出最终决策（PRIORITY/MODERATE/CAUTIOUS/REJECT）
3. 给出信心水平（高/中/低）

**输出格式**：
```
【综合评分汇总】
评分项          得分    权重    加权得分
工具接口设计    [得分]  30%     [加权得分]
数据格式统一    [得分]  20%     [加权得分]
错误处理机制    [得分]  20%     [加权得分]
性能优化        [得分]  20%     [加权得分]
安全性设计      [得分]  10%     [加权得分]
------------------------------------------------
综合评分        [综合评分]（满分10分）

【最终决策】
- 决策：PRIORITY / MODERATE / CAUTIOUS / REJECT
- 信心水平：高 / 中 / 低
- 集成建议：[建议描述]
```

**强制检查点**：必须给出明确的 PRIORITY/MODERATE/CAUTIOUS/REJECT 决策。

---

### 6.2 集成建议

**任务**: 给出具体集成建议。

**输出格式**：

```
- 集成优先级：[优先级]
- 集成时间：[时间估算]
- 集成成本：[成本估算]

- 集成收益：[收益描述]
```

**强制检查点**：必须给出集成优先级、时间、成本建议，缺失不得完成工作流。
