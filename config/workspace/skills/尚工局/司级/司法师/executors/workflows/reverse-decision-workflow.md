# 司法师反向决策工作流

> **执行者**: 司法师（Charlie Munger哲学）
> **用途**: 系统化反向决策流程
> **约束**: 必须完整执行，不得跳过任何步骤
> **强制读完**: 本文件包含 `[EXECUTION_COMPLETE]` 标记，AI必须读到该标记才算完成

---

## 工作流概述

本工作流基于 Charlie Munger 的反向思考哲学，采用 **失败路径识别 + 错误因素排除 + 逆向推理验证 + 多学科思维模型 + 风险预警机制** 五大核心方法论。

**执行原则**：
1. **强制读完机制**：AI必须执行完整流程，不得跳过任何步骤
2. **反向思考优先**：先思考如何失败，再思考如何成功
3. **多学科验证**：使用≥5个跨学科思维模型验证决策
4. **风险预警**：建立≥10个风险预警指标

---

## Phase 1: 决策目标明确（强制执行）

### 1.1 确定决策目标

**任务**: 明确决策目标和成功标准。

**执行步骤**：
1. 明确决策目标：
   - 决策类型（投资/产品/运营）
   - 决策目标
   - 成功标准
   - 时间期限

**输出格式**：
```
【决策目标】
- 决策类型：[类型]
- 决策目标：[目标描述]
- 成功标准：[标准描述]
- 时间期限：[期限]
```

**强制检查点**：必须明确决策目标，缺失不得进入下一阶段。

---

## Phase 2: 失败路径识别（强制执行）

### 2.1 识别失败路径

**任务**: 识别所有可能导致失败的路径。

**执行步骤**：
1. 头脑风暴失败路径（≥10个）
2. 分类失败路径：
   - 战略失败路径
   - 执行失败路径
   - 市场失败路径
   - 技术失败路径
   - 团队失败路径

**输出格式**：
```
【失败路径识别】
失败路径总数：[数量]个

战略失败路径：
1. [路径1]
2. [路径2]

执行失败路径：
1. [路径1]
2. [路径2]

市场失败路径：
1. [路径1]
2. [路径2]
```

**强制检查点**：
- 失败路径数 ≥ 10个：进入下一阶段
- 失败路径数 < 10个：**补充识别后继续**

---

### 2.2 失败路径评分

**任务**: 对失败路径进行严重性和概率评分。

**评分标准**（0-10分）：
- **严重性**：0-10分（10分=致命失败）
- **概率**：0-10分（10分=几乎必然发生）

**输出格式**：
```
【失败路径评分】
失败路径            严重性(0-10)  概率(0-10)  风险评分
[路径1]             [评分]        [评分]      [风险评分]
[路径2]             [评分]        [评分]      [风险评分]
------------------------------------------------
平均风险评分        [平均评分]
```

**强制检查点**：
- 平均风险评分 < 5分：进入下一阶段
- 平均风险评分 ≥ 5分：**立即终止，输出"不决策"决策**

---

## Phase 3: 错误因素排除（强制执行）

### 3.1 识别错误因素

**任务**: 识别可能导致决策错误的所有因素。

**执行步骤**：
1. 识别错误因素（≥10个）
2. 分类错误因素：
   - 认知偏差因素
   - 信息不足因素
   - 分析错误因素
   - 执行错误因素
   - 外部干扰因素

**输出格式**：
```
【错误因素识别】
错误因素总数：[数量]个

认知偏差因素：
1. [因素1]
2. [因素2]

信息不足因素：
1. [因素1]
2. [因素2]
```

**强制检查点**：
- 错误因素数 ≥ 10个：进入下一阶段
- 错误因素数 < 10个：**补充识别后继续**

---

### 3.2 错误因素排除

**任务**: 排除可能导致决策错误的所有因素。

**执行步骤**：
1. 制定排除方案
2. 验证排除效果
3. 记录排除结果

**输出格式**：
```
【错误因素排除】
错误因素            排除方案        排除效果
[因素1]             [方案]          [效果]
[因素2]             [方案]          [效果]
------------------------------------------------
排除成功率          [成功率]%
```

**强制检查点**：
- 排除成功率 ≥ 80%：进入下一阶段
- 排除成功率 < 80%：**补充排除后继续**

---

## Phase 4: 逆向推理验证（强制执行）

### 4.1 逆向推理路径

**任务**: 通过逆向推理验证决策的正确性。

**执行步骤**：
1. 设计逆向推理路径（≥3条）
2. 执行逆向推理
3. 对比推理结果

**输出格式**：
```
【逆向推理验证】
推理路径1：[路径描述]
- 推理结果：[结果]
- 一致性：[是/否]

推理路径2：[路径描述]
- 推理结果：[结果]
- 一致性：[是/否]

推理路径3：[路径描述]
- 推理结果：[结果]
- 一致性：[是/否]
```

**强制检查点**：
- 逆向推理一致性 ≥ 2/3：进入下一阶段
- 逆向推理一致性 < 2/3：**立即终止，输出"不决策"决策**

---

## Phase 5: 多学科思维模型验证（强制执行）

### 5.1 多学科思维模型应用

**任务**: 使用多学科思维模型交叉验证决策。

**执行步骤**：
1. 选择思维模型（≥5个）：
   - 经济学模型（供需、边际效用）
   - 心理学模型（认知偏差、行为经济学）
   - 物理学模型（临界点、熵增）
   - 生物学模型（进化论、生态位）
   - 数学模型（概率、统计）

2. 应用每个模型验证决策

**输出格式**：
```
【多学科思维模型验证】
思维模型            验证结果        一致性
经济学模型          [结果]          [是/否]
心理学模型          [结果]          [是/否]
物理学模型          [结果]          [是/否]
生物学模型          [结果]          [是/否]
数学模型            [结果]          [是/否]
------------------------------------------------
模型一致性          [一致性]%
```

**强制检查点**：
- 模型一致性 ≥ 60%：进入下一阶段
- 模型一致性 < 60%：**立即终止，输出"不决策"决策**

---

## Phase 6: 最终决策（强制执行）

### 6.1 反向思考评分汇总

**任务**: 汇总所有评分，生成最终决策。

**评分权重**：
- 失败路径识别：30%
- 错误因素排除：25%
- 逆向推理验证：25%
- 多学科思维模型：20%

**执行步骤**：
1. 计算综合评分（加权平均）
2. 给出最终决策（ACCEPT/MODIFY/REJECT）
3. 给出信心水平（高/中/低）

**输出格式**：
```
【综合评分汇总】
评分项              得分    权重    加权得分
失败路径识别        [得分]  30%     [加权得分]
错误因素排除        [得分]  25%     [加权得分]
逆向推理验证        [得分]  25%     [加权得分]
多学科思维模型      [得分]  20%     [加权得分]
------------------------------------------------
综合评分            [综合评分]（满分10分）

【最终决策】
- 决策：ACCEPT / MODIFY / REJECT
- 信心水平：高 / 中 / 低
- 决策建议：[建议描述]
```

**强制检查点**：必须给出明确的 ACCEPT/MODIFY/REJECT 决策。

---

### 6.2 风险预警机制

**任务**: 建立风险预警机制。

**输出格式**：
```
【风险预警机制】
预警指标            阈值          当前值        状态
[指标1]            [阈值]        [当前值]      [正常/预警]
[指标2]            [阈值]        [当前值]      [正常/预警]
[指标3]            [阈值]        [当前值]      [正常/预警]
```

**强制检查点**：必须建立≥10个风险预警指标，缺失不得完成工作流。
