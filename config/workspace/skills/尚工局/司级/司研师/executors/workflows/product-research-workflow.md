# 司研师产品研究分析工作流

> **执行者**: 司研师（Philip Fisher哲学）
> **用途**: 系统化产品研究与品质分析
> **约束**: 必须完整执行，不得跳过任何步骤
> **强制读完**: 本文件包含 `[EXECUTION_COMPLETE]` 标记，AI必须读到该标记才算完成

---

## 工作流概述

本工作流基于 Philip Fisher 的成长品质投资哲学，采用 **Scuttlebutt调研法 + 研发创新分析 + 管理质量评估** 三大核心方法论。

**执行原则**：
1. **强制读完机制**：AI必须执行完整流程，不得跳过任何步骤
2. **数据源唯一性**：所有金融数据必须使用 `neodata-financial-search` skill
3. **量化指标验证**：每个决策必须有量化指标支撑（研发投入≥5%营收、管理质量评分≥7分）
4. **Scuttlebutt验证**：必须完成客户访谈、供应商访谈、竞争对手分析

---

## Phase 1: 产品数据收集（强制执行）

### 1.1 获取目标公司产品数据

**任务**: 使用 `neodata-financial-search` skill 获取目标公司的产品相关数据。

**执行步骤**：
1. 调用 `neodata-financial-search` skill
2. 搜索目标公司的产品数据：
   - 产品名称
   - 产品类型
   - 市场份额
   - 用户数量
   - 研发投入
   - 产品迭代周期

**输出格式**：
```
【产品数据】
- 产品名称：[名称]
- 产品类型：[类型]
- 市场份额：[份额]%
- 用户数量：[数量]
- 研发投入：[投入]亿元
- 研发投入占比：[占比]%营收
- 产品迭代周期：[周期]个月
```

**强制检查点**：必须获取以上7项产品数据，缺失任何一项不得进入下一阶段。

## Phase 2: Scuttlebutt调研分析（强制执行）

### 2.1 客户访谈分析

**任务**: 分析客户对产品的反馈和满意度。

**访谈维度**：
1. **产品满意度**：客户对产品的整体满意度
2. **功能需求**：客户最需要的功能
3. **痛点问题**：客户使用产品的主要痛点
4. **竞品对比**：客户对竞品的看法

**输出格式**：
```
【客户访谈分析】
- 产品满意度：[满意度评分]/10
- 核心功能需求：[需求列表]
- 主要痛点：[痛点列表]
- 竞品对比：[对比分析]
```

**强制检查点**：必须完成客户访谈分析（至少10个客户样本）。

### 2.2 供应商访谈分析

**任务**: 分析供应商对公司的评价。

**访谈维度**：
1. **合作关系**：供应商与公司的合作稳定性
2. **付款信誉**：公司的付款及时性
3. **需求预测**：供应商对公司未来需求的预测

**输出格式**：
```
【供应商访谈分析】
- 合作关系评分：[评分]/10
- 付款信誉评分：[评分]/10
- 需求预测：[预测描述]
```

**强制检查点**：必须完成供应商访谈分析（至少5个供应商样本）。

### 2.3 竞争对手分析

**任务**: 分析竞争对手的产品策略。

**分析维度**：
1. **竞品数量**：主要竞争对手数量
2. **竞品优势**：竞争对手的核心优势
3. **竞品劣势**：竞争对手的主要劣势
4. **竞争格局**：市场竞争格局判断

**输出格式**：
```
【竞争对手分析】
- 竞品数量：[数量]
- 竞品优势：[优势列表]
- 竞品劣势：[劣势列表]
- 竞争格局：[格局描述]
```

**强制检查点**：必须完成竞争对手分析（至少5个竞品）。

## Phase 3: 研发创新分析（强制执行）

### 3.1 评估研发投入强度

**任务**: 评估公司的研发投入强度和创新能力。

**评估标准**：
- **优秀**：研发投入≥10%营收
- **良好**：研发投入≥5%营收
- **不足**：研发投入<5%营收

**输出格式**：
```
【研发创新分析】
- 研发投入：[投入]亿元
- 研发投入占比：[占比]%营收
- 研发人员数量：[数量]
- 研发人员占比：[占比]%员工总数
- 评估结果：[优秀/良好/不足]
```

**强制检查点**：必须评估研发投入强度并给出评估结果。

---

## Phase 4: 管理质量评估（强制执行）

### 4.1 评估管理团队质量

**任务**: 评估管理团队的能力和诚信度。

**评估维度**：
1. **战略能力**：管理团队的战略规划能力
2. **执行能力**：管理团队的执行效率
3. **创新能力**：管理团队的创新意识
4. **诚信度**：管理团队的诚信记录

**评分标准**（0-10分）：
- **优秀**：管理质量评分≥8分
- **良好**：管理质量评分≥7分
- **不足**：管理质量评分<7分

**输出格式**：
```
【管理质量评估】
- 战略能力评分：[评分]/10
- 执行能力评分：[评分]/10
- 创新能力评分：[评分]/10
- 诚信度评分：[评分]/10
- 管理质量总分：[总分]/10
- 评估结果：[优秀/良好/不足]
```

**强制检查点**：必须评估管理质量并给出评分。

## Phase 5: 产品竞争力综合评分（强制执行）

### 5.1 计算产品竞争力总分

**任务**: 综合所有分析维度，计算产品竞争力总分。

**计算公式**：
```
产品竞争力总分 = (市场份额 × 0.2) + (研发投入评分 × 0.3) + (管理质量评分 × 0.3) + (客户满意度 × 0.2)
```

**评分标准**（0-10分）：
- **优秀**：产品竞争力总分≥8分
- **良好**：产品竞争力总分≥7分
- **不足**：产品竞争力总分<7分

**输出格式**：
```
【产品竞争力综合评分】
- 市场份额评分：[评分]/10
- 研发投入评分：[评分]/10
- 管理质量评分：[评分]/10
- 客户满意度评分：[评分]/10
- 产品竞争力总分：[总分]/10
- 评估结果：[优秀/良好/不足]
```

**强制检查点**：必须计算产品竞争力总分并给出评估结果。

## Phase 6: 最终决策与风险提示（强制执行）

### 6.1 综合评估与决策

**任务**: 综合所有分析结果，给出最终产品投资建议。

**决策标准**：
- **BUY（投资产品）**：产品竞争力总分≥8分、研发投入≥5%营收、管理质量≥7分
- **HOLD（观察产品）**：产品竞争力总分≥7分、研发投入≥3%营收、管理质量≥6分
- **SELL（放弃产品）**：产品竞争力总分<7分、研发投入<3%营收、管理质量<6分

**输出格式**：
```
【最终决策】
- 决策：[BUY/HOLD/SELL]
- 产品竞争力总分：[总分]/10
- 研发投入占比：[占比]%营收
- 管理质量评分：[评分]/10
- 核心理由：[理由描述]

【操作建议】
- 产品策略：[策略建议]
- 研发策略：[策略建议]
- 管理策略：[策略建议]
- 竞争策略：[策略建议]
```

**强制检查点**：必须给出明确的BUY/HOLD/SELL决策和具体操作建议。

### 6.2 风险提示

**任务**: 提供产品投资的风险提示。

**输出格式**：
```
【风险提示】
- 主要风险因素：
  1. [风险1]
  2. [风险2]
  3. [风险3]

【免责声明】
本分析基于公开财务数据和市场调研信息，仅供参考，不构成任何投资建议。
产品投资有风险，决策需谨慎。过去的表现不代表未来的结果。
请结合自身投资目标和风险承受能力做出独立判断。
```
