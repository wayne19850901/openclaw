## 工作流概述

本工作流基于 Philip Fisher 的 Scuttlebutt 调研法，采用 **客户访谈 + 供应商访谈 + 竞品分析 + 行业专家访谈 + 实地调研** 五大核心方法论。

**执行原则**：
1. **强制读完机制**：AI必须执行完整流程，不得跳过任何步骤
2. **调研充分性**：客户访谈≥10人，供应商访谈≥5个，竞品分析≥5个
3. **信息真实性**：多源信息交叉验证
4. **调研深度**：访谈时长≥30分钟

---

## Phase 1: 调研准备（强制执行）

### 1.1 确定调研目标

**任务**: 确定产品开发的调研目标。

**执行步骤**：
1. 明确调研目标：
   - 产品名称
   - 产品功能
   - 目标用户
   - 目标市场

**输出格式**：
```
【调研目标】
- 产品名称：[名称]
- 产品功能：[功能描述]
- 目标用户：[用户描述]
- 目标市场：[市场描述]
```

**强制检查点**：必须明确调研目标，缺失不得进入下一阶段。

---

### 1.2 制定调研计划

**任务**: 制定详细的调研计划。

**执行步骤**：
1. 制定调研计划：
   - 客户访谈计划（≥10人）
   - 供应商访谈计划（≥5个）
   - 竞品分析计划（≥5个）
   - 行业专家访谈计划（≥3人）
   - 实地调研计划（≥3个地点）

**输出格式**：
```
【调研计划】
- 客户访谈：[数量]人，访谈时长≥30分钟
- 供应商访谈：[数量]个，访谈时长≥30分钟
- 竞品分析：[数量]个，分析深度≥高
- 行业专家访谈：[数量]人，访谈时长≥30分钟
- 实地调研：[数量]个地点，调研时长≥1小时
```

**强制检查点**：必须制定完整调研计划，缺失不得进入下一阶段。

---

## Phase 2: 客户访谈（强制执行）

### 2.1 执行客户访谈

**任务**: 执行客户访谈，了解用户需求。

**执行步骤**：
1. 访谈客户（≥10人）
2. 记录访谈内容：
   - 用户需求
   - 用户痛点
   - 用户期望
   - 用户使用场景

**输出格式**：
```
【客户访谈结果】
访谈人数：[数量]人
用户需求：
1. [需求1]
2. [需求2]
3. [需求3]

用户痛点：
1. [痛点1]
2. [痛点2]
3. [痛点3]

用户期望：
1. [期望1]
2. [期望2]
3. [期望3]
```

**强制检查点**：
- 访谈人数 ≥ 10人：进入下一阶段
- 访谈人数 < 10人：**补充访谈后继续**

---

### 2.2 客户访谈评分

**任务**: 对客户访谈进行评分。

**评分标准**（0-10分）：
- **9-10分**：访谈≥20人，访谈深度高，信息质量高
- **7-8分**：访谈≥15人，访谈深度中等，信息质量中等
- **5-6分**：访谈≥10人，访谈深度一般，信息质量一般
- **3-4分**：访谈<10人，访谈深度低，信息质量低
- **0-2分**：访谈失败，无法获取有效信息

**输出格式**：
```
【客户访谈评分】
- 访谈人数：[数量]人
- 访谈深度：[深度评分]/10
- 信息质量：[质量评分]/10
- 客户访谈总分：[总分]/10
```

**强制检查点**：
- 客户访谈总分 ≥ 6分：进入下一阶段
- 客户访谈总分 < 6分：**立即终止，输出"不开发"决策**

---

## Phase 6: 最终决策（强制执行）

### 6.1 Scuttlebutt调研评分汇总

**任务**: 汇总所有调研评分，生成最终决策。

**评分权重**：
- 客户访谈：30%
- 供应商访谈：20%
- 竞品分析：20%
- 行业专家访谈：20%
- 实地调研：10%

**执行步骤**：
1. 计算综合评分（加权平均）
2. 给出最终决策（PRIORITY/MODERATE/CAUTIOUS/REJECT）
3. 给出信心水平（高/中/低）

**输出格式**：
```
【综合评分汇总】
评分项          得分    权重    加权得分
客户访谈        [得分]  30%     [加权得分]
供应商访谈      [得分]  20%     [加权得分]
竞品分析        [得分]  20%     [加权得分]
行业专家访谈    [得分]  20%     [加权得分]
实地调研        [得分]  10%     [加权得分]
------------------------------------------------
综合评分        [综合评分]（满分10分）

【最终决策】
- 决策：PRIORITY / MODERATE / CAUTIOUS / REJECT
- 信心水平：高 / 中 / 低
- 产品开发建议：[建议描述]
```

**强制检查点**：必须给出明确的 PRIORITY/MODERATE/CAUTIOUS/REJECT 决策。

---

```
- 开发收益：[收益描述]
```

**强制检查点**：必须给出开发优先级、时间、成本建议，缺失不得完成工作流。
