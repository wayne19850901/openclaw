# 需求分析

## 概述

在开始任何创造性工作之前，探索用户意图、需求和设计。

## 核心原则

**必须在任何创造性工作之前使用**：创建功能、构建组件、添加功能或修改行为。

## 工作流程

```
1. 理解项目上下文
    ↓
2. 一次一个问题，逐步细化想法
    ↓
3. 理解要构建什么
    ↓
4. 展示设计并获取用户批准
    ↓
5. 开始实现
```

## 硬性规则

| 规则 | 原因 |
|------|------|
| 不调用实现工具 | 先理解再动手 |
| 不写代码 | 设计阶段不编码 |
| 不创建文件 | 等待批准后再创建 |

## 问题模板

1. 这个功能的主要用户是谁？
2. 他们想解决什么问题？
3. 成功的结果是什么样的？
4. 有什么技术约束？
5. 需要和哪些系统集成？
