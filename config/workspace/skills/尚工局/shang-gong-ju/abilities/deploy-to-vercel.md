# 应用部署

## 概述

部署应用到Vercel，获取预览链接。

## 核心原则

**总是部署为预览环境**，除非用户明确要求生产部署。

## 部署方式

| 方式 | 命令 |
|------|------|
| 预览部署 | `vercel deploy` |
| 生产部署 | `vercel deploy --prod` |

## 部署流程

```
1. 确认项目配置
    ↓
2. 运行部署命令
    ↓
3. 获取预览链接
    ↓
4. 验证部署成功
    ↓
5. 汇报链接
```

## 最佳实践

- 预览部署先于生产部署
- 检查环境变量配置
- 确认构建成功
- 验证功能正常
