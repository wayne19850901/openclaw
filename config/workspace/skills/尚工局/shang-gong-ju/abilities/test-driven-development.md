# TDD开发

## 概述

测试驱动开发：先写测试，看它失败，再写最小代码让它通过。

## 核心原则

**如果你没有看到测试失败，你不知道它是否测试正确的东西。**

## 工作流程

```
🔴 Red：写一个失败的测试
    ↓
🟢 Green：写最小代码让测试通过
    ↓
🔵 Refactor：重构代码
    ↓
重复
```

## 适用场景

| 场景 | 说明 |
|------|------|
| 新功能 | 总是TDD |
| Bug修复 | 先写失败测试 |
| 重构 | 确保测试覆盖 |
| 行为变更 | 先改测试 |

## 例外情况

- 抛弃式原型
- 生成代码
- 配置文件
