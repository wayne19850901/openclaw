# 系统调试

## 概述

系统化调试方法：总是先找到根本原因，再尝试修复。

## 铁律

```
没有根本原因调查，就没有修复
```

如果你还没有完成第一阶段，就不能提出修复方案。

## 调试流程

```
Phase 1: 问题定义
    ├── 描述预期行为
    ├── 描述实际行为
    └── 定义成功标准
    
Phase 2: 调查
    ├── 收集信息
    ├── 形成假设
    └── 验证假设
    
Phase 3: 修复
    ├── 实施修复
    └── 验证修复
```

## 常见错误

| 错误 | 正确做法 |
|------|----------|
| 随机修复 | 先找根本原因 |
| 快速补丁 | 解决根本问题 |
| 跳过调查 | 系统化调试 |
