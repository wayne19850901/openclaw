# 架构评审能力

> 整合自 gstack/plan-eng-review — Eng manager模式架构审查

## 概述

以工程经理视角审查架构方案，挑战范围合理性，评估复杂度与风险。

## 触发条件

- 变更涉及8+文件
- 新增2+类
- 架构层面改动
- 用户明确要求架构评审

## 执行流程

1. **加载完整指令**：读取 `~/.claw/workspace/skills/gstack/plan-eng-review/SKILL.md`
2. **范围挑战**：
   - 复杂度检查（8+文件 or 2+新类 → 需要拆分）
   - 范围蔓延检测
   - 依赖影响分析
3. **架构评审**：
   - 模块边界是否清晰
   - 依赖方向是否正确
   - 抽象层次是否合理
4. **代码质量检查**：
   - 命名规范
   - 代码组织
   - 重复代码
5. **测试覆盖分析**：
   - 覆盖率评估
   - 边界条件测试
   - 生成测试覆盖图
6. **性能影响评估**：
   - 潜在瓶颈
   - 资源消耗变化
7. **输出评审报告**

## 评审维度

| 维度 | 检查项 |
|------|--------|
| 范围 | 文件数、新类数、变更影响面 |
| 架构 | 模块边界、依赖方向、抽象层次 |
| 代码质量 | 命名、组织、重复、复杂度 |
| 测试 | 覆盖率、边界条件、集成测试 |
| 性能 | 瓶颈、资源消耗、可扩展性 |

## 输出格式

```
## 架构评审报告

### 范围评估
- 变更规模：[小/中/大]
- 是否需要拆分：[是/否]
- 理由：...

### 架构评估
- 模块边界：[清晰/模糊]
- 依赖方向：[正确/有问题]
- 建议：...

### 测试覆盖
- 覆盖率：XX%
- 缺失测试：...
- [测试覆盖图]

### 评审结论
APPROVE / NEEDS_REVISION / REJECT_WITH_RATIONALE
```
