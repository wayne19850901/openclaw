# 设计探索能力

> 整合自 gstack/design-shotgun — AI视觉变体生成与比较

## 概述

生成多个设计变体，通过比较看板收集反馈，迭代优化至最佳方案。

## 触发条件

- UI/UX设计需求
- 视觉风格探索
- 用户明确要求设计变体

## 执行流程

1. **加载完整指令**：读取 `~/.claw/workspace/skills/gstack/design-shotgun/SKILL.md`
2. **加载品味记忆**：
   - 读取用户历史偏好
   - 应用风格约束
3. **生成设计变体**（3-5个）：
   - 反收敛指令：确保变体之间差异显著
   - 每个变体独立风格方向
   - 标注设计理念
4. **创建比较看板**：
   - 并排展示所有变体
   - 启动HTTP反馈收集服务
   - 收集用户选择与评论
5. **迭代优化**：
   - 基于反馈调整
   - 可选：混合多个变体优点
6. **输出最终方案**

## 设计变体策略

| 策略 | 说明 |
|------|------|
| 极简主义 | 最少元素、最大留白 |
| 信息密度 | 紧凑布局、数据优先 |
| 视觉冲击 | 大胆配色、动态效果 |
| 经典稳健 | 传统布局、高可读性 |
| 实验创新 | 非常规布局、交互创新 |

## 品味记忆系统

记录用户偏好，用于后续设计：
- 颜色倾向
- 布局偏好
- 字体选择
- 交互风格
