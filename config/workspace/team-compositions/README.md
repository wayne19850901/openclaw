# 团队组合配置说明

## 概述

团队组合是 OpenClaw 系统的预定义协作场景，用于快速组建专业团队应对特定任务类型。每个团队组合包含明确的角色分工、工作流程和成功标准。

## 使用方式

### 1. 自动触发

当威帝的指令包含触发关键词时，Queen 会自动识别并激活对应的团队组合：

```
威帝："帮我快速做一个 MVP"
→ 自动触发 "Startup MVP 冲刺团队"

威帝："我们需要做内容营销"
→ 自动触发 "内容营销团队"
```

### 2. 手动指定

威帝也可以直接指定团队组合：

```
@Startup-MVP-Team 帮我开发一个产品原型
@Content-Marketing-Team 制定内容营销策略
```

### 1. Startup MVP 冲刺团队

**适用场景**：创业初期快速验证、MVP 开发、快速上线

**团队成员**：
- 慧妃（Leader）：技术架构 + 开发执行
- 仪妃：UI/UX 设计
- 昭妃：项目调度 + 进度把控
- 司证师：质量验证 + 风险审计

**工作流程**：慧妃开发 → 仪妃设计 → 昭妃调度 → 司证师验证

**预估时长**：2-4 周

**成功标准**：

- 技术债务可控

## 协作规则

每个团队组合都遵循以下通用协作规则：

1. **定期同步**：团队成员定期同步进度和问题
2. **质量审计**：质量审计贯穿全程
3. **效果验证**：关键节点进行效果验证
4. **持续改进**：基于反馈持续优化

## 扩展机制

团队组合支持动态扩展：

1. **临时加入**：根据任务需要，临时邀请其他 Agent 加入
2. **角色调整**：根据实际情况调整成员角色和职责
3. **流程优化**：根据执行效果优化工作流程

## 配置文件格式

每个团队组合配置文件为 JSON 格式，包含以下字段：

```json
{
  "name": "团队名称",
  "description": "团队描述",
  "trigger": ["触发关键词"],
  "leader": "团队领导",
  "agents": [
    {
      "id": "Agent ID",
      "role": "角色定位",
      "responsibilities": ["职责列表"]
    }
  ],
  "workflow": "工作流程",
  "success_criteria": ["成功标准"],
  "estimated_duration": "预估时长",
  "key_metrics": {
    "指标名": "目标值"
  },
  "collaboration_rules": ["协作规则"]
}
```
