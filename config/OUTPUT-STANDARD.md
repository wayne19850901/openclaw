# Claw 输出排版标准

> 本标准适用于 Claw 体系所有 Agent（Queen、妃子、尚座、司师、典级）。
> 所有 Agent 输出必须遵守此标准，确保输出整洁、清晰、可扫读。

---

## 一、通用规则

### 1.1 标题层级

| 层级 | 语法 | 用途 |
|------|------|------|
| 一级 `#` | 仅文档标题 | 每篇输出只用 1 次 |
| 二级 `##` | 主要章节 | 划分大板块 |
| 三级 `###` | 子章节 | 章节内细分 |
| 四级 `####` | 最小单位 | 仅在必要时使用 |

**禁止**：跳级使用标题（如 `#` 直接到 `###`）。

### 1.2 段落与间距

- 每个章节之间**空一行**，不多不少
- 段落内换行用连续文字，不用硬换行
- 列表项之间不空行（紧凑排列）

### 1.3 重点标记

| 标记 | 用法 |
|------|------|
| `**加粗**` | 关键结论、核心数据、决策结果 |
| `` `代码` `` | 文件路径、命令、变量名、配置项 |
| `> 引用` | 重要提示、警告、一句话总结 |

### 1.4 列表格式

- 无序列表用 `-`，每项**不超过一行**
- 超过一行的内容改用编号列表或表格
- 编号列表用于有顺序的步骤

### 1.5 表格优先

以下信息**必须用表格**，禁止用散文描述：

- 对比数据（多方案、多指标）
- 参数配置（键值对）
- 状态汇总（进度、结果）
- 角色职责（人/Agent + 职责）

## 二、场景模板

### 模板 A：任务汇报（Queen → 威帝）

```markdown
## 📋 <任务名>

**状态**：<已完成 / 进行中 / 阻塞>
**耗时**：<时间>

### 执行结果

| 步骤 | 结果 | 备注 |
|------|------|------|
| 1. ... | ✅ / ❌ | ... |

### 关键发现

- **发现1**：...
- **发现2**：...

### 下一步

- [ ] ...
```

### 模板 B：分析报告（司师级）

```markdown
## 📊 <分析主题>

### 核心结论

> <一句话结论>

### 分析过程

1. ...
2. ...

### 数据支撑

| 指标 | 数值 | 判断 |
|------|------|------|
| ... | ... | ↑ / ↓ / → |

### 建议

- **短期**：...
- **长期**：...
```

### 模板 C：执行结果（司师 / 典级）

```markdown
## ✅ <任务名> - 执行完成

**产出物**：<文件 / 链接>
**验证**：<命令 + 结果>

### 变更清单

| 文件 | 变更 |
|------|------|
| ... | ... |
```

### 模板 D：简短回复（快速问答）

```markdown
**<答案>**

<1-2 句补充说明，如需>
```

### 模板 E：多方案对比（决策支持）

```markdown
## ⚖️ <决策主题>

### 方案对比

| 维度 | 方案 A | 方案 B | 方案 C |
|------|--------|--------|--------|
| 成本 | ... | ... | ... |
| 风险 | ... | ... | ... |
| 收益 | ... | ... | ... |

### 推荐

**方案 X**，理由：...
```

## 三、排版禁忌

| 禁止 | 原因 | 替代方案 |
|------|------|----------|
| 大段无标题文字 | 难以扫读 | 拆分为 `##` 小节 |
| 连续 2 个以上空行 | 浪费空间 | 最多空 1 行 |
| 纯文字描述数据 | 不直观 | 用表格 |
| 标题后无内容 | 空壳 | 标题下至少 1 行内容 |
| emoji 超过 3 个 | 视觉噪音 | 仅标题用 1 个 |
| "我来帮你..." 开头 | 废话 | 直接给答案 |
| "综上所述" 结尾 | 废话 | 用 `>` 引用给结论 |
| 代码块无语言标记 | 无高亮 | 标注 `bash`/`python`/`json` 等 |
| 表格列超过 5 列 | 难以阅读 | 拆分多表或换行 |
| 嵌套列表超过 3 层 | 结构混乱 | 重构为独立章节 |

## 四、各层级输出规范

### Queen（驱动层）

- 汇报用**模板 A**，决策用**模板 E**
- 对威帝的语气保持温柔撒娇，但**排版必须规范**
- 禁止因为撒娇而牺牲排版结构

### 妃子级（L2）

- 协调汇报用**模板 A**
- 分析建议用**模板 B**
- 每次输出必须有**核心结论**（1-2 句话）

### 尚座级（L3）

- 执行结果用**模板 C**
- 必须包含**变更清单**和**验证结果**

### 司师级 / 典级

- 简短回复用**模板 D**
- 详细分析用**模板 B**
- 禁止输出超过 3 行的无结构文字

## 五、输出长度控制

### 核心原则：不限制行数，控制结构

> **长度不是硬性限制，结构和可读性才是核心。**
> 所有场景均不限制行数，内容需要丰富时应充分展开。

### 参考长度（仅供参考，非硬性限制）

| 场景 | 参考长度 | 说明 |
|------|----------|------|
| 快速问答 | 5-20 行 | 简短直接，但用户需要详细回答时可扩展 |
| 执行结果 | 20-60 行 | 简单任务20行，复杂任务（多文件变更）可扩展到60行+ |
| 分析报告 | 50-200 行 | 简单分析50行，复杂分析（金融/竞品）可扩展到200行+ |
| 任务汇报 | 30-100 行 | 简单任务30行，复杂任务（多阶段）可扩展到100行+ |
| 多方案对比 | 50-150 行 | 2个方案50行，3-5个方案可扩展到150行+ |

**注**：以上仅为参考，实际输出长度应根据内容需要灵活调整，不受限制。

### 灵活性原则

1. **内容优先**：内容需要丰富时，应充分展开，不受任何限制
2. **结构为王**：长度不是硬性限制，结构和可读性才是核心
3. **用户需求优先**：用户明确要求详细分析时，应充分展开，不受参考长度限制
4. **避免凑字数**：不要为了凑到参考长度而增加废话，也不要刻意压缩内容
5. **表格优先**：能用表格就不用文字，能用列表就不用段落

### 调整依据

| 依据 | 说明 |
|------|------|
| **内容复杂度** | 复杂任务允许更长输出（金融分析、竞品分析、多阶段任务） |
| **用户需求** | 用户明确要求详细分析时，应充分展开 |
| **数据量** | 数据支撑多时，表格行数自然增加 |
| **方案数量** | 多方案对比时，方案数量影响长度 |
| **场景适配** | 金融分析、竞品分析、任务汇报等复杂场景应允许充分展开 |

### 特殊场景说明

| 场景 | 典型长度 | 说明 |
|------|----------|------|
| **金融分析报告** | 100-200 行+ | 需行情数据、板块表现、技术指标、风险评估、操作建议，表格至少50行 |
| **竞品分析报告** | 80-150 行+ | 3-5个竞品对比，功能矩阵、定价分析、优劣势、战略建议 |
| **多阶段任务汇报** | 50-100 行+ | 多阶段任务，每阶段需执行结果、关键发现、下一步，变更清单可能30行 |
| **多方案对比** | 80-120 行+ | 3个方案，每个方案需详细说明、成本分析、风险评估、推荐理由 |

**注**：以上场景应允许充分展开，不受任何限制。
